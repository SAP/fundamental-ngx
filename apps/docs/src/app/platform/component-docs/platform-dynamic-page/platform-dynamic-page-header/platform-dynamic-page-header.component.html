Dynamic Page
The dynamic page is the foundation for all pages in SAP Fiori. It is a generic layout designed to support various use cases; there, the content of both the header and page can vary.

The header of the dynamic page is collapsible, which helps the users focus on the actual page content but still ensures that important header information and actions are readily available.


The basic structure of Dynamic Page is:
  • Title+Header - comprises of Title and Header area where Header is collapsible.
    The Title area(fdp-dynamic-page-title) allows user to input title and subtitle, and in turn can hold user-specified content in breadcrumbs(through fd-breadcrumb), KPI content(fdp-dynamic-page-key-info), global actions(through fdp-dynamic-page-global-actions), layout actions(through fdp-dynamic-page-layout-actions).
    The header content is projected through fdp-dynamic-page-header and allows user to specify the options:
    • collapsible- whether the header can be collapsed. True by default. If set to false, both pin/collapse buttons disappear and the header stays visible.
    • collapsed- the initial state of the header. Set to true if header should be collapsed.
    • pinnable- whether the header should be allowed to be pinned or unpinned. When set, the pin button shows up. Pinning the header will make the header stay visible and the collapse button(if present) will disappear until unpinned.
  • Content - Content is projected through fdp-dynamic-page-content. User can use multiple fdp-dynamic-page-contents with tabLabel specified to hook a tab navigation internally and provide tabbed content.
  • Footer - Content is projected through fdp-dynamic-page-footer. Use a floating footer from the Bar component.

Other features of the header are:
  • Expand/Collapse Header Feature - The expand/collapse header feature is a small icon button directly below the header content that enables the user to expand or collapse the header content. When the pin is active, the expand/collapse header feature is still available and overrules the pin.
  • Snap on Scroll - Scrolling is the typical trigger for collapsing the header content. When the user scrolls down the page, the header content scrolls away. The header title switches to collapsed mode, and stays fixed. When the user scrolls up the page, the header content expands again.
  • Snap on Click - In addition to the “snap on scroll” function, the user can also expand and collapse the header content by clicking anywhere on the header title bar that has no active elements (such as links or active buttons).
  • Pinning the Header Content - Activating the pin feature keeps the header expanded while the user scrolls through the page content. This mode remains fixed until the user clicks the pin icon again. This feature is optional and can be switched off if it is not useful for your application.

Platform specific enhancements:

  • Automatic generation of tab navigation and control of multiple content panels.
