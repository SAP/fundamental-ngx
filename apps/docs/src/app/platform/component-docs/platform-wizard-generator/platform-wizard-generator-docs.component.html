Default Wizard generator The default wizard displays a walkthrough screen with forms split into sections. Once the user fills all of the necessary input fields, and presses the "Next step" button, new step will appear. A summary page can be created by adding the [addSummary]="true" input to the wizard generator component. Wizard generator in dialog The wizard generator can also be used within a dialog by calling open method from WizardDialogGeneratorService A Cancel button is displayed in the footer, so the user can exit the wizard at any point during the process. However, if a user exits the wizard after adding their input, they will receive a warning that their information will be lost. Customizable Wizard generator The wizard component can be customized with modifier inputs. Steps can be placed on separate pages, rather than appended to the wizard, by setting the [appendToWizard] input to false.

You can also use icons for the step indicators by adding icon property to the wizard steps object.

Buttons appearance can be changed with [navigationButtonLabels] property.

Dialog close confirmation text can be changed with confirmationDialogText, confirmationDialogCloseText, confirmationDialogCancelText properties.

If you want to use completely different style for navigational buttons, you can use goNextButtonTemplate, goBackButtonTemplate, finishButtonTemplate, cancelButtonTemplate properties for the dialog wizard, and content-projected html with special directives: fdpWizardGeneratorGoNextButton, fdpWizardGeneratorFinishButton.

If you want to use your custom confirmation dialog, you can pass it confirmationDialogTemplate property to wizard dialog service. Branching Wizard Developers can configure branching behaviour of the wizard by providing when property function to the wizard generator step and by specifying dependencyFields property of the wizard generator step to reflect which steps are creating branches. Responsive paddings for Wizard generator

The default wizard dont have responsive paddings applied for progress bar and content. To enable responsive paddings, you can use [responsivePaddings]="true" input to the wizard generator component.
