Default Wizard generator The default wizard displays a walkthrough screen with forms split into sections. Once the user fills all of the necessary input fields, and presses the "Next step" button, new step will appear. A summary step can be added by providing additional step with summary: true property to the wizard generator metadata. Wizard generator in dialog The wizard generator can also be used within a dialog by calling open method from WizardDialogGeneratorService A Cancel button is displayed in the footer, so the user can exit the wizard at any point during the process. However, if a user exits the wizard after adding their input, they will receive a warning that their information will be lost. Customizable Wizard generator The wizard component can be customized with modifier inputs. Steps can be placed on separate pages, rather than appended to the wizard, by setting the [appendToWizard] input to false.

You can also use icons for the step indicators by adding icon property to the wizard steps object.

Buttons appearance can be changed with [navigationButtonLabels] property.

If you want to use completely different style for navigational buttons, you can use special directives: fdp-wizard-generator-go-next-button, fdp-wizard-generator-review-button, fdp-wizard-generator-finish-button.

To change Summary step view, you can use fdp-wizard-generator-summary-step directive to flag your template as a summary section template. Your template will receive such variables:
  • steps: Array of object that implements WizardGeneratorSummaryItem interface
  • editStep: Method to edit defined section step
 Customizable Wizard generator in dialog Dialog close confirmation text can be changed with confirmationDialogText, confirmationDialogCloseText, confirmationDialogCancelText properties.

If you want to use completely different style for navigational buttons, you can use goNextButtonTemplate, goBackButtonTemplate, reviewButtonTemplate, finishButtonTemplate, cancelButtonTemplate properties for the dialog wizard.

If you want to use your custom confirmation dialog, you can pass it confirmationDialogTemplate property to wizard dialog service. Branching Wizard Developers can configure branching behaviour of the wizard by providing when property function to the wizard generator step and by specifying dependencyFields property of the wizard generator step to reflect which steps are creating branches. Responsive paddings for Wizard generator

The default wizard have responsive paddings applied for progress bar and content. To disable responsive paddings, you can use [responsivePaddings]="false" input to the wizard generator component.

Visible summary step in progress bar By default, Wizard generator doesn't show Summary step in the wizard progress bar section. Developers can configure visibility of the summary step by providing [displaySummaryStep]="true" property of <fdp-wizard-generator />. Visible summary step with branching As per design specification, no steps should be visible after branching step, until it's completed. This means that developers should configure appropriate condidions for steps visibility. Same applies to the summary step, if it should be visible in wizard progress bar. Responsive paddings for Wizard generator in dialog Use responsivePaddings: false property for dialog config to dosable wizard responsive paddings inside dialog. Handling complex objects rendering on Summary page If your form field contains some complex object and should be rendered with custom function, you can provide valueRenderer function for particular form item. External navigation buttons If you want navigation buttons of the Wizard generator to be outside the component, you can use public methods and properties of the Wizard generator itself to implement desired behaviour.
