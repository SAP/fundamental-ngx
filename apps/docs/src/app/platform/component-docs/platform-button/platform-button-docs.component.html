Button Options There are three emphasis styles used to indicate the importance of the button on the page. Do not provide a value for the default option. Use [options]="'emphasized'" or [options]="'light'" to set the option. Button Types The default button type is "action", no modifier is required. Use [type]="'standard'", [type]="'positive'", [type]="'medium'", or [type]="'negative'" to modify the type of the button. Button Sizes Use [compact]="true" for the mobile button size. Button with Icons Use [glyph]="'{{ '{' }}icon-name}'" to add an icon to the button. Button State [disabled]="true" makes the button disabled, when we make it false, it won't be disabled any more, by default disabled is set to false. Button with long text truncated When user specifices with width of the button then text will start truncating to accomdate it self in the specified width as shown below The inputs for button will be translated into classes attributed to fundamental-styles guidelines.
For example:
if positive is passed for type, the class fd-button--positive will be added to the element.
if compact is passed for size, the class fd-button--compact will be added to the element. Extensibility The button implementation allows certain extensibility in terms of options for the inputs. button inputs don't have a restriction for the predefined values.

You can pass a number of custom options to the options input, which accepts as a parameter either a string or an array of strings.

For example, [options]="'emphasized'" will add the class fd-button--emphasized to the button.

If you'd like to add multiple options, you could use [options]="['emphasized', 'option1', 'option2']".
This would result in 3 classes being added to the button: fd-button--emphasized, fd-button--option1, and fd-button--option2.
