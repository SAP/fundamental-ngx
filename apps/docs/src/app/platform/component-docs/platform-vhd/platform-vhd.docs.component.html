Basic setup

fdp-value-help-dialog
  • Set [dialogTitle], it should be string. It will show as header's title of the dialog window.
  • Set [uniqueKey] it should be uniq field from data source. Required field for `selected from list` tab
  • Set [tokenViewField] or [tokenizerFn] property to render selected item token. Required field for `selected from list` tab [tokenViewField] should be a string equal key from data source. Need to render selected token.
    [tokenizerFn] should be a function. It has higher prio that `tokenViewField`
  • Set [dataSource] property to your data source. Required field for `selected from list` tab
fdp-value-help-dialog-filter
  • Set [key] property to string, should be equal to field from data source. Required field.
  • Set [label] property to string, it will show as header in search table. If you skip this field, key will appear instead label.
Custom strategy labels.

Example to demonstrate custom strategy labels for Define Conditions tab

By default we're using strategy labels:

  • contains = 'contains'
  • equalTo = 'equal to'
  • between = 'between'
  • startsWith = 'starts with'
  • endsWith = 'ends with'
  • lessThan = 'less than'
  • lessThanEqual = 'less than or equal to'
  • greaterThan = 'greater than'
  • greaterThanEqual = 'greater than or equal to'
  • empty = 'empty'

Set [defineStrategyLabels] for you custom strategy label. All keys are optional

  • Allowed keys: contains, equalTo, between, startsWith, endsWith, lessThan, lessThanEqual, greaterThan, greaterThanEqual, empty
Token display function for the Value Help Dialog.
  • Set [tokenViewField] property to name from data source fields. It'll use for render custom token in selection panel.
  • Set [tokenizerFn] property to your function. It'll use for render custom token in selection panel.

[tokenizerFn] has higher priority than [tokenViewField]

Example to demonstrate the token display function for selected items panel in the footer of the Value Help Dialog.

 Filters combinations

All filters will appear in advanced, includes and excludes sections by default.

You can control appearing with options

  • Set [advanced] property to false, if you want to hide filter from advanced search panel
  • Set [includes] property to false, if you want to hide filter from includes panel in Dedine Conditions Tab
  • Set [excludes] property to false, if you want to hide filter from excludes panel in Dedine Conditions Tab

Example to demonstrate combinations with filters

 Selection in Value Help Dialog.
  • Set [searchSelection] property to multi, if you want to select multiple rows from search table
  • Set [searchSelection] property to single, if you want to select single row from search table without close dialog
  • Set [searchSelection] property to once, if you want to select single row from search table and success dialog with this value
