Search Field The data for the typeahead dropdown list can be provided via the suggestions property. Data provided this manner can either be an array of strings or an Observeable with an array of strings as output. Suggestion data will be filtered and highlighted based on the user input. The Search Field component outputs two types of events: an inputChange event whenever the input text changes, and a searchSubmit event when the user types a keyboard enter or clicks the search icon button. The placeholder text for the Search Field can be changed with the placeholder property. The Search field size property can be either cozy or compact The Search field ariaLabel property can be used to set ARIA attribute to clarify what search field is. The Search field ariaLabelledBy property can be used to set ARIA attribute with id of elements which labels search input to clarify what search field is. Search Field with Categories Categories can be added to the search field by assigning an array of strings to the categories property. The selected category is included in the event data emitted in both the inputChange and searchSubmit events. Search Field with Data Source A dynamic dataSource can be provided to the Search Field as an alternative to the static string array provided via the suggestions property. Use the SearchFieldDataSource class and SearchFieldDataProvider base class to create a data source which can be connected to a back-end service.
