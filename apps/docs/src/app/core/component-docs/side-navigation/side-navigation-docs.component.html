Side Navigation with One Level List It's mandatory to use [textOnly]="true" to fd-nested-list, when there are no icons. Side Navigation with Titles Use this to group navigation. Titles are not clickable. Side Navigation with Compact Mode Add [compact]="true" to switch to compact mode Side navigation with multiple levels Use this when there is more than one level of hierarchy in the left navigation. Each level can be expanded and collapsed. Side navigation with Multiple Levels and Selected Option Side navigation with Programmatically State Change. Side Nav also allows to control the expanded and selected state, from outside. Side navigation with Icons To put icons into the Side Navigation, there is no need to add [textOnly]="true" to fd-nested-list anymore. Side Navigation Condensed Mode For Mobiles, or lower screen devices there is condensed mode. It's recommended to use it with icons, because icons are the only elements, that are visible in first level navigation. Also nested lists should be placed in the fd-nested-list-popover element. Side Navigation Generated from Configuration Object Side Navigation component can be generated by configuration object SideNavigationModel, by passing options and content. Side Navigation Condensed Mode Generated from Configuration Object Side Navigation component generated by configuration object can be used along with condensed mode. The popover is added automatically.
