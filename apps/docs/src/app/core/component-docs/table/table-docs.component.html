Default Table Table with Toolbar The application developer can use the toolbar alongside the table to provide custom functionality to the user. In this example, the user can add new items to the table and perform a simple local search on all data in the table. Table Without Borders The [noBorderX] and [noBorderY] inputs can be used to remove borders. They can be used together or separately, and can be added to fd-table, fd-table-header, fd-table-body, and fd-table-cell components. Table with Footer Footer can be added by using the fd-table-footer directive. Customizable Columns The user can be given greater control over what appears in the table if the application developer provides an array of [keys] as an input to the table component. Each cell must have a key provided that corresponds to one of the values in the [keys] array. In this example, the user can press the "Open Table Customization" button to open a modal. Each of the [keys] is displayed in a list and the user can use the checkboxes to decide what columns will appear in the table, and they can also set the order of the columns using the arrow up/arrow down buttons. Column Sorting and Filtering In this example, the first column can be sorted alphabetically or filtered based on text in an input. Click the first column header to try. Interactive Rows and Cells Interactive states can be set by using the [activable] and [hoverable] inputs on rows and cells. Focusable Table rows and cell can be focusable, by adding [focusable]="true" and tabindex="0" to it. Table with Checkboxes The checkbox input can be used at the beginning of each row to allow for bulk actions. Table With Semantic Row Highlighting To show that an item needs attention, you can show a highlight indicator next to the item. This is done using the fd-table-status-indicator directive, along with the [type] input. Acceptable values for this input are valid, warning, information, and error. Table with Angular CDK The fd-table directive can be used in conjunction with the Angular CDK, granting the functionality of the CDK table with the style of the Fundamentals table. CDK's Drag and Drop feature can be used along with the Fundamental Library for Angular table. You can rearrange rows, columns, or cells, subscribe to events and handle data changes in your component accordingly. Pop In Mode on Responsive Table Table has a mode to be support mobile devices. It's called Pop In mode, it can be achieved by adding [popIn]="true" and changing a markup. One default row should be divided to 2 rows, main and secondary. First(main) one is interactive, second has to be used only to display rest of information. More important data has to go to first row. Amount of cells is limited to minimum, to improve overall look of table.
To achieve responsive table, you need to enable responsive mode by adding [responsive]="true" to fd-table. Table with navigatable rows To represent that the row is navigatable put icon or button to the last column and set [activable] and [hoverable] properties of the row Hide navigation icons for the non-navigatable rows. Disable navigation buttons for the non-navigatable rows. Upon navigation event, either the user is taken to a new page, or the event is handled by an implementation such as (click)="myFunction()" Table With Pagination
