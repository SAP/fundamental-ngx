Simple Tabs Simple tabs can be built by setting specified properties to fd-tab. There is way to define own templates, or use it with [routerLink]. There is no compact mode on inline tabs. Tabs with Counter Tabs with counter can be built by setting specified properties to fd-tab. There is way to define own templates, or use it with [routerLink]. There is no compact mode on inline tabs with counter. Icon Only Mode The Icon Only mode can be used, by adding [mode]="'icon-only'". It requires different structure usage. Tabs in Icon Only mode can be built by setting specified properties to fd-tab. There is way to define own templates, or use it with [routerLink]. Process Mode The Process Mode can be used by adding [mode]="'process'". As on icons it has different structure than tab any other mode. It can be built by setting properties, defining own template, or using it with [routerLink]. Filter Mode The Filter Mode can be used by adding [mode]="'filter'". It has different structure than tab any other mode. It can be built by setting properties, defining own template, or using it with [routerLink]. To make filter Mode Tabs looks properly, it's mandatory to add [header]="true" property to first tab item, or in case of custom template use different structure, as it is shown in example. Programmatic Selection There are two ways of accomplishing this. First, the selectedIndex input can be used. This requires maintaining a flag. The second method requires setting a template reference variable and then calling the selectTab(index) method.
