Select Modes Following examples presents different modes of the Select control. Select Semantic States Use semantic states to visually highlight current state of the Select. Custom Control Content The content of the Select control can be customized using a custom template which needs to be referenced in the [controlTemplate] input. Select provides the following context values to the template:
  • Text value of selected option - implicit
  • Reference to selected option - selected
Selected option as a context to custom template. Extended Options ListComponent features such as headers, titles and icons are supported inside the OptionComponent. This example also shows how to handle complex option types like Object. Select In Mobile Mode To use Select in mobile mode:
  • import SelectMobileModule
  • Set [mobile] property to true
  • Set [mobileConfig] with MobileModeConfig configuration object to customize the look of the control
Select Mobile Mode can be configured globally by providing MOBILE_MODE_CONFIG token. Check Global Configuration page for more information. Select Max Height The Select component allows you to define the [maxHeight]. It can be useful when there are a lot of options given that the select body is be scrollable. By default the maximum height of the Select body is 45% of the browser's inner window height. Adding and Removing Options Adding and removing options dynamically is supported seamlessly with very little extra work required from the developer. By design, if the user removes the option that is currently selected, the value will become undefined. This was implemented to prevent a user from keeping the active value of a previously removed option. This behavior can be automatically prevented by setting [unselectMissingOption]="false". Programmatic Control Controlling Select through programmatic methods is possible by calling the Select component instance directly. Note that this can be done through ViewChild as well. Select In A Reactive Form The Select component can be used inside Angular reactive forms.
