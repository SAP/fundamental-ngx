Select composition Select is composed out of:
  • SelectComponent - main component representing input control
  • OptionComponent - component representing select option
Select states Examples of Select control in different states Select semantic states Custom control content Content of Select control can be customized with custom template. Custom template reference must be provided to [controlTemplate] input. Selects provide selected option as a context to custom template. Extended Options OptionComponent is based on ListComponent thus list element features like headers, titles and icons are supported inside select option. This example also shows how to handle options that are complex types like Object Custom View Value The value which appears inside select control can be easily customized using OptionComponent [viewValue] input. Select Max Height The select component allows to define [maxHeight]. It can be useful when there are a lot of options, the select body is be scrollable. By default maximum height of select body is 45% of browsers window inner height. Adding and Removing Options Adding and removing options dynamically is supported seamlessly, with very little extra work required from the developer. By design, if the user removes the option that is currently selected, the value will become undefined. This was implemented to prevent a user from keeping the active value of a previously removed option. Also user can prevent automatically removing missing option by passing [unselectMissingOption]="false". Programmatic Control Controlling the select through programmatic methods is possible by calling the select component instance directly. Note that this can be done through ViewChild as well. Select In A Reactive Form The select component can be used inside Angular reactive forms.
