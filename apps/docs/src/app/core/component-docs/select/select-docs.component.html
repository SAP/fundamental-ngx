Select modes Select control is composed out of two components: SelectComponent and OptionComponent.
Below you can find examples of Select control in different modes. Select semantic states Custom control content Content of Select control can be customized with custom template. Custom template reference must be provided to [controlTemplate] input.
Select provides following context values to the template:
  • Text value of selected option - implicit
  • Reference to selected option - option
selected option as a context to custom template. Extended Options OptionComponent is based on ListComponent thus list element features like headers, titles and icons are supported inside select option. This example also shows how to handle options that are complex types like Object. Custom View Value The value which appears inside select control can be easily customized using OptionComponent [viewValue] input. Select in mobile mode To open Menu in mobile mode compose the Menu Component together with the Dialog Component.
There are a few important things to remember:
  • Because of Select being tightly bind with Option it is important to keep the options in view without instantiating and destroying them with every dialog open/close action. We can easily achieve it by creating the Dialog once and using DialogRef.hidden() method to show/hide the dialog.
  • To get full Fiori3 experience set Select in mobile mode using [mobile]="true". Also remember to instantiate the Dialog inside the Select using DialogConfig.container and Select.dialogContainerElementRef.
  • Options template needs to be declared inside fd-select tags to properly configure element injectors. Use Select.selectOptionsListTemplate to render options in desired place.
Select Max Height The select component allows to define [maxHeight]. It can be useful when there are a lot of options, the select body is be scrollable. By default maximum height of select body is 45% of browsers window inner height. Adding and Removing Options Adding and removing options dynamically is supported seamlessly, with very little extra work required from the developer. By design, if the user removes the option that is currently selected, the value will become undefined. This was implemented to prevent a user from keeping the active value of a previously removed option. Also user can prevent automatically removing missing option by passing [unselectMissingOption]="false". Programmatic Control Controlling the select through programmatic methods is possible by calling the select component instance directly. Note that this can be done through ViewChild as well. Select In A Reactive Form The select component can be used inside Angular reactive forms.
