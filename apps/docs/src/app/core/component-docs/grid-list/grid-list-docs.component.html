Modes Use the [mode] property to specify the one of these modes.

Modes:

  • none: Default mode (no selection)
  • delete: Delete mode (only one list item can be deleted via provided delete button)
  • singleSelect: Selected item is highlighted but no selection control is visible (only one list item can be selected)
  • singleSelectLeft: Left-positioned single selection mode (only one list item can be selected)
  • singleSelectRight: Right-positioned single selection mode (only one list item can be selected)
  • multiSelect: Multi-selection mode (more than one list item can be selected)
 None mode Items cannot be selected but can still use "navigation", which allows click handling on specific items. Single select mode One item in the grid list can be selected. Need click on an item to select it. To use the Single select mode, set [mode] as singleSelect. Single select left mode One item in the grid list can be selected. For this, the grid list provides radio buttons on the left side of each item toolbar. To use the Single select left mode, set [mode] as singleSelectLeft. Single select right mode One item in the grid list can be selected. For this, the grid list provides radio buttons on the left side of each item toolbar. To use the Single select right mode, set [mode] as singleSelectRight. Multi select mode Users can select one or more items. For this, the grid list provides checkboxes on the left side of each item toolbar. To use the Multi select mode, set [mode] as multiSelect. Delete mode This places a Delete button on the right side of an item toolbar. To use the Delete mode, set [mode] as delete. Group If grouped, a group header is displayed above all items which belong to the corresponding group. The group header is not interactive. To create grid list group headers use fd-grid-list-group-header component. States To apply state, set [state] as unread|error|locked|draft. Status A status indicator is shown in front of the item it needs to show that an item needs attention. To add status indicator, set [status] as success|warning|error|neutral. More button The More button loads more items to the front end if not all items have yet been loaded. Footer The footer can be used to display additional static information relating to the content. Drag and Drop Layout Defines the column layout in the format XLn-Ln-Mn-Sn where n is the number of columns and can be different for each size.
Default: XL4-L3-M2-S1
For a base Layout, we can use only 1, 2, 3, 4, 6, or 12 columns in a row.
But it can be more flexible using layoutItemPattern. Focusing Item Programmatically Use focus() method of the GridListItemComponent to focus item programmatically.
