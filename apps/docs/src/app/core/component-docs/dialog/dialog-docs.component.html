Template based Dialog The Dialog can be declared inside ng-template and opened with DialogService using template reference. Each template based dialog can use provided context values:
  • DialogRef - implicit
  • DialogConfig - dialogConfig
DialogRef and DialogConfig reference needs to be passed to Dialog component when using template based Dialog. To receive Dialog dismiss/close reasons subscribe to afterClosed observable. Component based Dialog To open a component based Dialog provide component class to the DialogService open method. Inject DialogRef dependency inside the component to access Dialog methods and data. Data available in DialogRef can be provided in DialogConfig object passed as an argument to DialogService.open() method. Dialog state Dialog state can be controlled with DialogRef, providing such features like:
  • DialogRef.hide() - visually hiding the Dialog window
  • DialogRef.close() - closing the Dialog window
  • DialogRef.dismiss() - dismissing the Dialog window
  • DialogRef.loading() - displaying the dialog in loading state
 Dialog configuration Dialog behavior can be configured using DialogConfig object passed as an argument to DialogService.open() method. Passing DialogConfig overrides specified behaviours of the Dialog component. Default values used by the Dialog can be specified by providing DIALOG_DEFAULT_CONFIG token value.

@NgModule({{'{'}} provide: [ {{'{'}}provide: DIALOG_DEFAULT_CONFIG, useValue: customConfig{{'}'}} ] {{'}'}} Dialog positioning The placement of the Dialog on the screen can be customized through the DialogConfig.position property. Dialog on mobile devices To display Dialog component in mobile mode set use DialogConfig.mobile property. Dialog on mobile devices takes full viewport and adjusts its header and footer. To create additional spacing around Dialog use DialogConfig.mobileOuterSpacing property.

To achieve full-screen dialog without mobile mode features use DialogConfig.fullScreen property. Complex Dialog To create more complex Dialogs it is possible to provide custom header, subheader and footer templates.
To provide custom template create element with on of the following directives:
  • Header - fdTemplate="header"
  • Subheader - fdTemplate="subheader"
  • Footer - fdTemplate="footer"
Custom templates are embedded views for Bar Component so it is possible to use Bar Component features inside them. Stacked Dialogs The DialogService supports limitless stacked dialogs. Each can be closed individually, there is no specific closing order.

In the example below, it is accomplished via components as content. However, it is entirely possible with templates through the same methodology. Custom Backdrop and Container The dialog backdrop and container can be styled, disabled or even customized at will. Custom css classes can be applies using:
  • DialogConfig.containerClass
  • DialogConfig.backdropClass
  • DialogConfig.dialogPanelClass
The Dialog can be appended to whatever DOM element you choose. Simply pass an HTMLElement to the DialogConfig.container configuration option. Note that there is no visual distinction between parents. This functionality can be used to apply a select set of style rules to a specific dialog, for example.
