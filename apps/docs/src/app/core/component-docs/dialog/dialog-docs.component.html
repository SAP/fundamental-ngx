Template based Dialog The Dialog can be declared inside the ng-template and opened with DialogService using the template reference. Each template based dialog can use the following context values:
  • DialogRef - implicit
  • DialogConfig - dialogConfig
The DialogRef and DialogConfig references need to be passed to the Dialog component when using template based Dialog. To receive Dialog dismiss/close reasons subscribe to the afterClosed observable.
  • Use fd-dialog-footer-button to wrap footer buttons and create space between them.
  • Add fd-dialog-decisive-button to key dialog buttons like "Save" or "Cancel" to add proper styles and initial focus.
 Component based Dialog To open a component based Dialog provide the component class to the DialogService open method. Inject the DialogRef dependency inside the component to access Dialog methods and data. Data available in DialogRef can be provided in the DialogConfig object and passed as an argument to the DialogService.open() method. Dialog state The dialog state can be controlled by using DialogRef, providing such features as:
  • DialogRef.hide() - visually hiding the Dialog window
  • DialogRef.close() - closing the Dialog window
  • DialogRef.dismiss() - dismissing the Dialog window
  • DialogRef.loading() - displaying the dialog in loading state
 Dialog configuration Dialog behavior can be configured using the DialogConfig object passed as an argument to the DialogService.open() method. Passing DialogConfig overrides specified behaviours of the Dialog component. The default values used by the Dialog component can be specified by providing the DIALOG_DEFAULT_CONFIG token value.

Dialog on mobile devices To display the Dialog component in mobile mode, use the DialogConfig.mobile property. Dialog on mobile devices uses the full viewport and adjusts its header and footer accordingly. To create additional spacing around the Dialog, use the DialogConfig.mobileOuterSpacing property.

To achieve full-screen dialog without mobile mode features, use the DialogConfig.fullScreen property. Complex Dialog To create more complex Dialogs it is possible to provide custom header, subheader and footer templates.
To provide these custom templates, create an element with one of the following directives:
  • Header - fdTemplate="header"
  • Subheader - fdTemplate="subheader"
  • Footer - fdTemplate="footer"
Custom templates are embedded views for the Bar Component so it is possible to use Bar Component features inside them.
For complex templates you may use the fd-bar-element as an equivalent of the fd-dialog-footer-button. Stacked Dialogs The DialogService supports limitless stacked dialogs. Each can be closed individually and there is no specific closing order.

In the example below, this is accomplished by using components as content however this can also be achieved by using templates. Custom Backdrop and Container The dialog backdrop and container can be styled, disabled or even customized at will. Custom css classes can be applied using:
  • DialogConfig.containerClass
  • DialogConfig.backdropClass
  • DialogConfig.dialogPanelClass
The Dialog can be appended to whatever DOM element you choose. Simply pass an HTMLElement to the DialogConfig.container configuration option. Note that there is no visual distinction between parents.
