Basic This example shows simple splitter usage. Basic input properties.

fd-splitter:

  • Use [defaultPaneId] input property of the fd-splitter to define main pane. Please make sure the default pane is in the root container (not in nested container).

fd-splitter-pane-container:

  • use orientation to set pane orientation. May be vertical (default) or horizontal.
  • listen for (resize) event to get resize information (panes and their updated widths).

fd-splitter-split-pane:

  • Use [size] input property of the fd-splitter-split-pane to define pane width.
  • Use [id] input property of the fd-splitter-split-pane to define pane id.
 Required parent width
  • Use [requiredParentWidth] (number, pixels) input property of the fd-splitter-split-pane to set minimal required window's width when pane will be shown.
  • Use [demandPane] (boolean) to specify if the pane will be available using the pagination bar when it's off-canvas.
API of the Splitter & subcomponents fd-splitter has next API methods:
  • splitter.showPaneOnCanvas(paneId: string) to show certain pane on the canvas.
  • splitter.hidePaneFromCanvas(paneId: string) to hide certain pane from the canvas.
  • splitter.isPaneOnCanvas(paneId: string) to check whether certain pane is on canvas.
fd-splitter-split-pane has next API methods & getters:
  • splitPane.showOnCanvas() to show pane on the canvas.
  • splitPane.hideFromCanvas() to hide pane from the canvas.
  • splitPane.isOnCanvas to check whether pane is on canvas.
