Standard Checkbox By default fd-checkbox component is binary, returning true and false values. Tristate Checkbox Tristate mode adds third state to default checkbox behaviour. In this mode checkbox returns: true, false and null values.
To prevent user from manually selecting third state of the checkbox use [tristateSelectable] property. Checkbox custom values Values returned by checkbox can be customized using [values] property.
  • Type: {{ '{ trueValue?: any, falseValue?: any, thirdStateValue?: any }' }}
  • Default value: {{ '{ trueValue: true, falseValue: false, thirdStateValue: null }' }}
Checkbox with Reactive Forms Checkbox can be used with Reactive Forms by binding it as standard form control with [formControlName] input. Example below shows usage of checkbox component with Reactive Form. "Accept all" behaves as main control representing overall state of agreements controls. Checkbox with custom label Checkbox label can by passed to component as external content using content projection mechanism. If external content is provided value passed using [label] property is ignored. Checkbox styling properties Checkbox can be but in various states by modifying following properties:
  • Modifies visual appearance of checkbox
    [state] = 'success' | 'error' | 'information' | 'warning'
  • Disables control
    [disable] = true | false
  • Miniaturizes checkbox
    [compact] = true | false
