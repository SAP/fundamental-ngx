Default Wizard The default wizard displays a walkthrough screen with forms split into sections. Once the user fills all of the necessary input fields, a button is displayed to go to the next section. A Cancel button is displayed in the footer, so the user can exit the wizard at any point during the process. However, if a user exits the wizard after adding their input, they will receive a warning that their information will be lost.

The wizard does not have any built-in functionality for moving through each step - the application developer will manage this using the [status] input on each fd-wizard-step

By default, the wizard content's height will be the viewport height, minus the height of the shellbar, wizard header and wizard footer. If the application developer wishes to use a custom height, a CSS value can be passed to the [contentHeight] input.

A summary page can be created by adding the [isSummary] input to the final wizard step. The summary page is optional, it is up to the developer to create this page. Customizable Wizard The wizard component can be customized with modifier inputs. Steps can be placed on separate pages, rather than appended to the wizard, by setting the [appendToWizard] input to false. Mobile Wizard For mobile devices, use the fd-bar component with the barDesign="floating-footer" modifier to achieve a floating footer. The Next Step button is placed after the content whenever possible. However, it can be placed above the floating footer by adding the [floating] input to the fd-wizard-next-step component. Branching Wizard A branching step connector can be used by applying the [branching] boolean input to the wizard step. This should be used if one step may lead to two different steps afterwords. In this example, step 2 may take a user to either a credit card information page or a bank transfer information page. Dialog Wizard The wizard can also be placed in a dialog. The [appendToWizard] input must be set to false, and a dialog width and height must be provided to ensure the dialog does not change size as the user progresses through steps and to the summary page. It is recommended that width and height are set to 100% but any value may be provided. Wizard with steps in ngFor Alternatively, wizard steps can be created by iterating over an array using ngFor. Visible summary step in progress bar By default, Wizard component doesn't show Summary step in the wizard progress bar section. Developers can configure visibility of the summary step by providing [displaySummaryStep]="true" property of <fd-wizard />.
