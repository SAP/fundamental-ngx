Default Time Picker Some default settings such as [displaySeconds], [displayMinutes], [displayHours] and [meridian] are based on the DateTimeFormats.display.timeInput options.
For example if time picker format option contains hours, minutes and day period info then default options will be the next: [displaySeconds]="false", [displayMinutes]="true", [displayHours]="true" and [meridian]="true". The listed above options are applied to the Time component only and can be defined explicitly if needed. Time parsing/formatting There are several ways to define your specific options to format and parse time.
  1. Provide your own DATE_TIME_FORMATS and define display/parse timeInput option there.
  2. Use time picker [parseFormat] and [displayFormat] options. Please note that you must provided these options in a format that is understandable by DateAdapter that is currently used in your app.
Disabled Time Picker Set [disabled]="true" to disable the time picker. Compact Time Picker Set [compact]="true" to use compact time-picker. Null Validity Null input values are considered valid by default. Use [allowNull]="false" to make them invalid. Time Picker in Form The time-picker component may also be used together with Angular reactive forms. Time Picker Locale The time-picker component is listening to the DateTimeAdapter locale, this locale can be changed dynamically.
