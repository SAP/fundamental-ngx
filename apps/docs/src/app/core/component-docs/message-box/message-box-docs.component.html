Object based message box The easiest and fastest way to create a simple message box is to use object based approach. Message box content can be declared as a MessageBoxContent and passed as an argument to MessageBoxService. To create a message box with more complex content use template reference as MessageBoxContent.content. Template based message box The message box can be declared inside the ng-template and opened with MessageBoxService using the template reference. Each template based message box can use the following context values:
  • MessageBoxRef - implicit
  • MessageBoxConfig - messageBoxConfig
The MessageBoxRef and MessageBoxConfig references need to be passed to the message box component when using template based message box. To receive message box dismiss/close reasons subscribe to the afterClosed observable.
  • Use fd-message-box-footer-button to wrap footer buttons and create space between them.
  • Add fd-message-box-decisive-button to key buttons like "Save" or "Cancel" to add proper styles.
  • By default after opening the message box first focusable element will be focused. To focus specific element after opening the message box, use fd-initial-focus directive.
Component based message box To open a component based message box provide the component class to the MessageBoxService open method. Inject the MessageBoxRef dependency inside the component to access message box methods and data. Data available in MessageBoxRef can be provided in the MessageBoxRef object and passed as an argument to the MessageBoxService.open() method. Semantic types Message box supports various semantic types to visually highlight the type of the message. Use MessageBoxConfig.type to create various types of message box. Message box positioning The placement of the Message box on the screen can be customized through the MessageBoxConfig.position property.

Message Box can be configured globally by providing MESSAGE_BOX_DEFAULT_CONFIG token. Check Global Configuration page for more information. Message box on mobile devices To display the Message box component in mobile mode, use the MessageBoxConfig.mobile property. Message box on mobile devices uses the full viewport and adjusts other elements for smaller screens. To create additional spacing around the Message box in mobile mode, use the MessageBoxConfig.mobileOuterSpacing property. Complex template To create more complex message box it is possible to provide custom header, subheader and footer templates. To provide these custom templates, create an element with one of the following directives:
  • Header - fdTemplate="header"
  • Subheader - fdTemplate="subheader"
  • Footer - fdTemplate="footer"
Custom templates are embedded views for the Bar Component so it is possible to use Bar Component features inside them. For complex templates you may use the fd-bar-element as an equivalent of the fd-message-box-footer-button.
