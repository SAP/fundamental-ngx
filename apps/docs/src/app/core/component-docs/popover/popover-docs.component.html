Simple Popovers Making use of simple popovers is very easy. Open an fd-popover component, place a control and body inside. You are not required to keep track of the popover state. Popover with Header and Footer Popover body can be created with header and footer components. To add header, there can be added fd-popover-body-header component, for footer there is dedicated fd-popover-body-footer component. Popover Placement Popper.js offers a variety of placements for users to choose from. These can be any of 'top', 'bottom', 'right', 'left', and then can be suffixed with '-start' or '-end'. Programmatic Control Controlling the popover through programmatic means is easy. Simply two-way bind the isOpen property and change it. There are also some methods such as open() and close() which can be called through reference variable. Popovers in Dialogs Popovers are functional inside dialogs. The default behaviour is to overflow the dialog, though that can be changed with the options input, which accepts Popper.js settings. Fill Control Width A custom Popper modifier allows the directive to match its popover size to that of the control. Dynamic Body Height In case of height change inside popover body, the position of popover is not automatically changed, mostly cause of optimization reasons. To force change of the position, it is needed to call updatePopover() method inside popover, as it is on example. There is used SideNavigation component. Dropdown Popover Dropdown is a composition of basic popover and button with [fdMenu]="true".

Under the hood, the Popover component makes use of PopoverDirective. It abstracts some functionality in favor of consistency, but it may not be sufficient for every use case. If more flexibility is required, we recommend the use of the directive which can be found here.
