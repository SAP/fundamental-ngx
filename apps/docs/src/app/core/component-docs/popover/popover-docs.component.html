Basic Popovers Making use of simple popovers is very easy. Open an fd-popover component, place a control and body inside. You are not required to keep track of the popover state. Simple Popover There is different way to build popover, it can be done, by using simplified markup, with trigger element connected to which connected is popover. Popover with Header and Footer Popover body can be created with header and footer components. To add header, there can be added fd-popover-body-header component, for footer there is dedicated fd-popover-body-footer component. Popover Placement Popover offers a variety of placements for users to choose from. These can be any of 'top', 'bottom', 'right', 'left', and then can be suffixed with '-start' or '-end'. Programmatic Control Controlling the popover through programmatic means is easy. Simply two-way bind the isOpen property and change it. There are also some methods such as open() and close() which can be called through reference variable. Popovers in Dialogs Popovers are functional inside dialogs. Fill Control Width A custom Popper modifier allows the directive to match its popover size to that of the control. Different Popover Container Different element to which the popover body is attached can be added, by passing [appendTo]="elementRef". Otherwise the popover body is rendered out of markup, in < body >. Popover Focus Trap To apply focus trap, you need to add [focusTrapped]="true". To move focus automatically into popover, when it's opened, you need to apply [focusAutoCapture]="true". After closing popover, focus will go to latest active element. Popover CDK Placement There is different way of putting placement, which is accepted by CdkOverlay. [cdkPositions] is the property that accepts array of objects with positions related to overlay(popover-body) and origin(popover-control).
  • For horizontal (overlayX, originX), those values are 'start' | 'center' | 'end'.
  • For vertical (overlayY, originY), those values are 'bottom' | 'center' | 'top'.


This property accepts array, just to deal with situation, when overlay body goes beyond the screen. Popover Scroll Example There is also way to change scroll behaviour, CDK Overlay offers 4 different options
  • CloseScrollStrategy - popover is closed, upon scrolling.
  • NoopScrollStrategy and BlockScrollStrategy - popover stays in same place.
  • RepositionScrollStrategy(default) - popover follows the element, to which is appended to. There is way to apply another argument, to close popover, when it goes beyond the screen and add throttle time for performance purposes.
Dynamic Body Height In case of height change inside popover body, the position of popover is not automatically changed, mostly cause of optimization reasons. To force change of the position, it is needed to call updatePopover() method inside popover, as it is on example. There is used SideNavigation component. Dropdown Popover Dropdown is a composition of basic popover and button with [fdMenu]="true".
