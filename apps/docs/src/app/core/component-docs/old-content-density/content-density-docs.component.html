Content Density can be maintained app-wide using the ContentDensityService, and components can subscribe to a BehaviorSubject managed by the service and use that to handle changes to the content density. Before using it, you should add ContentDensityService to providers array to AppModule, or other root module in your application.
Note that many components only come in one size, and others (such as the button below) only have compact and cozy sizes, so setting the size to condensed will have same effect as compact.
All components that have content density options will first check to see if the developer has provided a compact input value, and if not, it will then refer to the value set in the ContentDensityService.
