Overflow Layout Standard Usage

Overflow Layout calculates available space based on the summed width of the elements with the applied fdOverflowLayoutItem and *fdOverflowItemRef directives.

If the width of the parent container is not sufficient to fill all items, Overflow Layout will automatically create a "More" button with the rest of the items inside a popover component.

If items inside Overflow Layout should be focusable, developers can pass [focusable]="true" input property.

Overflow Layout reacts to the main element resize and to each item resize to fit items without user interaction.

Additionally, developers can pass maximum visible items amount via [maxVisibleItems] input property.

Forcing visibility of the items

Developers can force visibility of the item if needed.

To force the visibility, set [forceVisibility] input property to true.

 Overflow Layout Complex example

Developers are able to define own "More" button template and it's logic with *fdOverflowExpand directive.

This directive returns array of hidden items with their template references.
