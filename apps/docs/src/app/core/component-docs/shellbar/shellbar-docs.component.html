Basic This example shows the minimum shellbar for a single application product with only user settings. If no user thumbnail is available then display initials. Links with collapsible menu and product switcher. When a product has multiple links, the product links should collapse into an overflow menu on mobile screens. All actions, except for the user menu, should be collapsed. See the placement of the <fd-shellbar-collapse> container below. Shellbar with SideNavigation There is used sideNavigation in the Shellbar component. To add some control button, it is mandatory to use fd-shellbar-side-nav, which will be placed in the left side of shellbar. How it works
  • The shellbar should be placed inside the shell layout container.
  • The shellbar is fluid and responsive by default with proper margins and padding built in.
  • The shellbar actions are duplicated into the overflow menu on mobile screens.
  • Product menu data gets passed as inputs to the <fd-product-menu> component. The user, user menu and product switcher are passed as inputs to the <fd-shellbar-actions> component. Data for each action gets passed as inputs to the <fd-shellbar-action> component. The response to clicks on any of these items is handled via callback functions for each object. See the 'collapsible' example to gain an understanding of how this works.
