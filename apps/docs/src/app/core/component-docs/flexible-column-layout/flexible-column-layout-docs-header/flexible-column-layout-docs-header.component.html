Flexible Column Layout
The flexible column layout is a layout control that displays multiple floorplans on a single page. This allows faster and more fluid navigation between multiple floorplans than the usual page-by-page navigation. The flexible column layout offers different layouts with up to three columns. Users can expand the column they want to focus on, switch between different layouts, and view the rightmost column in full screen mode.

Depending on the current layout and display size, the flexible column layout consists of one, two, or three horizontally-aligned columns. Each column contains content that is not provided by the flexible column layout itself. This content is passed to the component through ng-template and using the template references #startColumn, #midColumn and #endColumn.

Possible Layouts

  • full screen layouts
    • start column in full screen - (start: 100%, mid: 0, end: 0) - 'OneColumnStartFullScreen'
    • middle column in full screen - (start: 0, mid: 100%, end: 0) - 'OneColumnMidFullScreen'
    • end column in full screen - (start: 0, mid: 0, end: 100%) - 'OneColumnEndFullScreen'
  • 2-column layouts
    • start column expanded - (start: 67%, mid: 33%, end: 0) - 'TwoColumnsStartExpanded'
    • middle column expanded - (start: 33%, mid: 67%, end: 0) - 'TwoColumnsMidExpanded'
    • end column expanded - (start: 0, mid: 33%, end: 67%) - 'TwoColumnsEndExpanded'
  • 3-column layouts
    • middle column expanded - (start: 25%, mid: 50%, end: 25%) - 'ThreeColumnsMidExpanded'
    • end column expanded - (start: 25%, mid: 25%, end: 50%) - 'ThreeColumnsEndExpanded'
    • start column minimized - (start: 0, mid: 67%, end: 33%) - 'ThreeColumnsStartMinimized'
    • end column minimized - (start: 33%, mid: 67%, end: 0) - 'ThreeColumnsEndMinimized'

Size LG and XL (Desktop)

Possible variations - 'OneColumnStartFullScreen', 'OneColumnMidFullScreen', 'OneColumnEndFullScreen', 'TwoColumnsStartExpanded', 'TwoColumnsMidExpanded', 'ThreeColumnsMidExpanded', 'ThreeColumnsEndExpanded', 'ThreeColumnsStartMinimized' and 'ThreeColumnsEndMinimized'.

Size MD (Tablet)

There is no 3-column layout due to the limited width. However, you can still load 3 pages in size MD. Instead of showing them all side-by-side, the user can switch between columns 1/2 and 2/3 using 'ThreeColumnsStartMinimized' and 'ThreeColumnsEndMinimized'.

Possible variations - 'OneColumnStartFullScreen', 'OneColumnMidFullScreen', 'OneColumnEndFullScreen', 'TwoColumnsStartExpanded', 'TwoColumnsMidExpanded', 'TwoColumnsEndExpanded', 'ThreeColumnsStartMinimized' and 'ThreeColumnsEndMinimized'.

Size SM (Phone)

Because of the limited width, there is no multi-column layout for size SM. Instead, the rightmost column is shown in full screen mode.

Possible variations - 'OneColumnStartFullScreen', 'OneColumnMidFullScreen' and 'OneColumnEndFullScreen'.

Note: The column separators cannot be used to expand a column to full screen mode. The layout arrow is located next to the divider and points in the direction in which a column can be expanded. If a column cannot be expanded any further, the action is hidden.

The user cannot change the size of a column freely (for example, (start: 15%, mid: 85%, end: 0)).

For more information about Layouts visit the Fiori Design Guidelines.

The navigation through the flexible column layout (forward and backwards), as well as the actions for entering and exiting full screen mode and closing columns are not provided automatically by the flexible column layout control, and need to be implemented manually. This can be done by reacting to events, such as button click, and updating the [layout] property of the fd-flexible-column-layout with the desired value. The same [layout] property is used to specify the inital layout of the component.

For example, setting [layout]="'TwoColumnsMidExpanded'" will render a 2-column layout with middle column expanded.

For available navigation and close sequences visit the Fiori Design Guidelines.

Responsiveness

The flexible column layout changes its behavior for sizes LG, MD and SM in real time whenever the user resizes the screen. For more information visit the Responsiveness section of Fiori Design Guidelines.

By default, the Flexible Column Layout applies the Fiori 3 break points:

  • Size SM: max-width: 960px
  • Size MD: min-width: 961px and max-width: 1280px
  • Size LG/Size XL: min-width: 1281px

The component also provides a default behaviour for managing the screen resizing.
To overwrite the default breakpoints use the smBreakPoint and lgBreakPoint properties. It's also possible to provide a custom function that will handle the resizing of the screen. Use the customOnResizeFunction property to overwrite the default behaviour.

The backgroundDesign property specifies the background design of the column layout. The default is set to "solid". The other two options include "translucent" and "transparent".
