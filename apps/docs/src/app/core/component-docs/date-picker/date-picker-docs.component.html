Simple Date Picker You can use two-way data binding to select a date or range of dates on a calendar. Calendars internally utilize an interface called CalendarDay, which contains the property date. To select a date you will create a new object that contains the property date, whose value is a date instance which DatetimeAdapter implements. Then, using two-way binding syntax, set the selectedDay value to your new date. Note that selecting a day (or range of days) in this manner will override any block/disable functions. The default format for the input is provided via DateTimeFormats. See the 'Formatting' section for details on how to change this. The value for the input only changes the model on blur or an enter/return keystroke. Range Date Picker Allows the selection of a range of dates. Only the first and last dates are passed back under ngModel. Internationalization To use another locale it is needed to provide translated aria labels and additional locale data if it's required by used DatetimeAdapter. Formatting Providing a custom format for the dates is possible through providing DATE_TIME_FORMATS config. Note that when providing a custom format, the [placeholder] input should be set to match the new format as well. The dateFormat pipe can also be utilized to convert the date into a string according to the provided format. Null Validity Null input values are considered valid by default. Use [allowNull]="false" to make them invalid. Date Picker in Reactive Form The date-picker component may also be used within Angular Reactive Forms. The [disableFunction] controls the valid flag.
