Button Behaviors A MenuItemComponent can be passed to the [selected] input to select a menu item. A button title and callback function can be provided as an object through the [mainAction] input. This object can contain three properties, mainActionTitle, keepMainAction and callback, the first of which is a string for the button title, the second a boolean for whether or not the mainAction should not change when another menu option is selected, and the third is a function that will be called when the button is clicked. keepMainAction and callback are optional. This title and the corresponding button action will be changed when another option in the menu is selected, unless keepMainAction is set to true. If no [mainAction] is provided, the text and functionality of the first menu item will be set on load. By default, the main action button will keep its original width on load, regardless of the length of the text of the selected option. To have the button width change with the text length, set [fixedWidth]="false". Button Types

The default button type is "action", no modifier is required.

To change button type, use one of these options for fdType input property:

  • standard
  • emphasized
  • ghost
  • positive
  • negative
  • attention
  • transparent

Leave fdType="" empty to use Action button.

 Buttons with Icons Use glyph="icon-name" to change an icon of dropdown menu. Programmatic Selection and Open / Close It is possible to control the open state of the popover using MenuComponent.open() and MenuComponent.close() methods. A menu option can also be selected programmatically using the [selected] input. Custom Template for Action Button It is possible to add a custom template to the action button through the fd-split-button-action-title directive.
