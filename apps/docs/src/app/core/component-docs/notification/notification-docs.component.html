Notification

Notification Component can be used with a lot of different types and sizes. Also there is example of container usage. It is possible to provide container reference, where should the notification belong. By default it is a body element.

Template as Content

The NotificationService allows passing in a template reference as the content of the notification. Add let-notification to the template element to access the injected NotificationRef.

The example also features a confirmation notification. Subscribe to afterClosed to access the returned value. A dismissal will trigger an error on the observable, similar to rejecting a promise. The example notification is dismissed after 4 seconds.

Component as Content

A component type can be passed to the NotificationService open method as the content of the notification. Inject NotificationRef inside the content component to access methods such as close and dismiss.

Notification Group

Items are added to a list based on internal events within apps or systems. The list can display various types of notifications, which can differ in terms of appearance, content, and functionality. The list can be ordered by Date (the most recent notification appears first), by Type (notifications appear as grouped notifications ordered from A-Z) and by Priority (from highest to lowest priority).
