Message Page

Message page displays a message to the user when an app or list is empty or if an error has occurred. The text and icon on the message page changes depending on the use case, however, the layout stays the same. It is responsive and adjusts to fit the available screen space.

Usage

The message page can be used in the following situations:
  • Filter with no results
  • Search with no results
  • Empty app
  • Too many items
  • Item saved as a tile that is no longer available
  • Error

Structure

  • fd-message-page - the user can specify the type of the Message Page through the type input property which can be set to 'filter', 'search', 'no-items' and 'error'. The type will automatically assign an icon for the Message Page. If no type is specified the default icon is set to 'documents'. The icon can be customized with the glyph input property. In this case leave the type input property unset. It's recommended to always show an icon with the Message Page.   • fd-message-page-title - a mandatory element.
  • fd-message-page-subtitle - an optional element that provides additional details to the title. It should be as concise as possible. The text can formatted (bold, italic, underline and/or bullet points).
  • fd-message-page-actions - an optional container for buttons that represent actions associated with the message. It's highly recommended not to add more than two buttons.
  • fd-message-page-more- optional. A container for more actions.
