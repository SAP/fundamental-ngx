initialise

  • initialise(triggerElement, config?, templateData?): void

  • Method initialising the popover service - should be called, after view is initialised, params:

    • triggerElement - element, which is equivalent for cdkOverlayOrigin, or fd-popover-control, by default to this element the popover body will lbe appended and events on this element will trigger popover's toggle
    • config - configuration object using PopoverConfig interface
    • templateData - in case of having already PopoverBodyComponent, there is way to pass container, template containing PopoverComponent and PopoverComponent instance
