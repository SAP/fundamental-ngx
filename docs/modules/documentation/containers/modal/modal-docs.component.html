Modal
The modal is used as a container to be displayed in response to a action. It is commonly used to collect simple information with a short form, to get confirmation or display contextual information that does not require a page.

<fd-modal>

Template as Content

The ModalService allows passing in a template reference as the content of the modal. Add let-modal to the template element to access the injected ModalRef.

The example also features a confirmation modal. Subscribe to afterClosed to access the returned value. A dismissal will trigger an error on the observable, similar to rejecting a promise.

Component as Content

A component type can be passed to the ModalService open method as the content of the modal. Inject ModalRef inside the content component to access methods such as close and dismiss.

The example also features a modal with a scrollable body.

Fullscreen Modal

The modal dimensions can be controlled through the width and height properties of the configuration object. These accept strings, so you may use whatever unit works best for your situation.

Stacked Modals

The ModalService supports limitless stacked modals. Each can be closed individually, there is no specific closing order.

In the example below, it is accomplished via components as content. However, it is entirely possible with templates through the same methodology.

Backdrop Configuration

The modal backdrop can be styled, disabled or even customized at will. Classes being applied to the backdrop must be in the global styles file.
