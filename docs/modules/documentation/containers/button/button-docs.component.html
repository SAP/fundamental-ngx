Button
The Button directive is used to enhance standard HTML buttons.

Buttons can be navigated to via the tab key and pressed with the space bar or enter key.

<button fd-button>

The inputs for button will be translated into classes attributed to fiori-fundamentals guidelines.
For example:
if positive is passed for fdType, the class fd-button--positive will be added to the element.
if compact is passed for size, the class fd-button--compact will be added to the element.

Extensibility

The button implementation allows certain extensibility in terms of options for the inputs. button inputs don't have a restriction for the predefined values.
For example, passing the value "someState" to the state input will result in the class fd-button--someState being added to the button element. You can pass a number of custom options to the options input, which accepts as a parameter either a string or an array of strings. For example, [options]="'emphasized'" will add the class fd-button--emphasized to the button. If you'd like to add multiple options, you could use [options]="['emphasized', 'option1', 'option2']". This would result in 3 classes being added to the button: fd-button--emphasized, fd-button--option1, and fd-button--option2.

Button Options

There are three emphasis styles used to indicate the importance of the button on the page. Do not provide a value for the default option. Use [options]="'emphasized'" or [options]="'light'" to set the option.

Button Types

The default button type is "action", no modifier is required. Use [fdType]="'standard'", [fdType]="'positive'", [fdType]="'medium'", or [fdType]="'negative'" to modify the type of the button.

Button Sizes

Use [compact]="true" for the mobile button size.

Buttons with Icons

Use [glyph]="'{{ '{' }}icon-name}'" to add an icon to the button.

Button State

Use [state]="'disabled'" or [state]="'selected'" to set button state.
