Icon
Icons are used throughout the UI to save space, allow for visual clarity and focus, and for fun. Icons can be used adaptively if desired, but at this point they are used more as visual elements within other components.

<fd-icon>

The inputs for fd-icon will be translated into classes attributed to fiori-fundamentals guidelines.
For example:
if it is passed accept for glyph the class sap-icon--accept will be added to the element.
if it is passed xs for size the class sap-icon--xs will be added to the element.

Extensibility

The icon implementation allows certain extensibility in terms of options for the inputs. fd-icon inputs doesn't have a restriction for the predefined values.
The predefined values for the input size are xs, s, l, xl, but size can accept any other string, for example xxs, which will be translated into class sap-icon--xxs.
