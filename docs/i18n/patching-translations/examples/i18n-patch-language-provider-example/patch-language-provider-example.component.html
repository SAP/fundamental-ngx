When to Use Provider vs. Directive

Use patchLanguage() Provider When:
  • ✅ Multiple components need same patches
  • ✅ Want to patch entire module/route
  • ✅ Cleaner code (no template clutter)
  • ✅ Patches are static/known at compile time
Example use cases:
  • Multi-tenant apps (per-customer wording)
  • Feature modules with consistent terminology
  • Locale variants (US vs. UK English)
Use [fdPatchLanguage] Directive When:
  • ✅ Only one specific element needs patching
  • ✅ Dynamic patches based on template data
  • ✅ Conditional patches (if/else)
  • ✅ Component-specific overrides
Example use cases:
  • Context-specific wording for same component
  • User preference-based translations
  • A/B testing different copy
💡 Pro Tip: You can use both! Apply patchLanguage() for broad changes and [fdPatchLanguage] for specific overrides. The directive takes precedence over the provider.
