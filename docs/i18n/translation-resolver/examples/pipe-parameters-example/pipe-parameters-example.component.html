Pipe Parameters: This example demonstrates real-world usage of the fdTranslate pipe with object parameters, default values, and null handling. Use the controls below to see how translations update dynamically.

1. Basic translation (no parameters):

{{ ('coreDatePicker.dateInputLabel' | fdTranslate)() }}

{{ "{{ ('coreDatePicker.dateInputLabel' | fdTranslate)() }}" }}
2. With object parameter (dynamic count):

{{ ('coreMultiInput.countListResultsPlural' | fdTranslate: { count: itemCount() })() }}

Item Count: {{ itemCount() }}
{{ "{{ ('coreMultiInput.countListResultsPlural' | fdTranslate: { count: itemCount() })() }}" }}
3. With multiple parameters:

Simulated: "Welcome {{ userName() }}! You have {{ itemCount() }} items in your cart."

{{ "{{ ('welcome.message' | fdTranslate: { name: userName(), count: itemCount() })() }}" }}
4. With default fallback (for missing translations):

{{ ( 'nonExistent.translationKey' | fdTranslate: {} : 'This is the fallback text because the key does not exist' )() }}

{{ "{{ ('nonExistent.key' | fdTranslate: {}: 'Fallback text')() }}" }}
5. Null key handling (for optional translations):

Promo code message: {{ (promoCode() | fdTranslate: {} : 'No active promo code')() }}

{{ "{{ (promoCode() | fdTranslate: {}: 'No promo code')() }}" }}
6. In attribute bindings (aria-label, title, etc.): {{ '<button [attr.aria-label]="(\'coreButton.save\' | fdTranslate)()"></button>' }}

Important Notes

  • Always invoke with (): The pipe returns Signal<string>, so you must call it like a function.
  • Parameters are reactive: When signal values change (like itemCount()), the translation automatically updates.
  • Default values are optional: Use the third parameter to provide fallback text for missing translations.
  • Null keys are safe: Passing null/undefined as the key will use the default value instead of throwing an error.
  • Works everywhere: Use in interpolation, attribute bindings, property bindings, etc.
💡 Pro Tip: For translations you use multiple times, consider using resolveTranslationSignal() in your component instead. It's more efficient than repeating the pipe in multiple places.
