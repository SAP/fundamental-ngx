Standard Select

A basic example of the select component with no advanced features.

Programmatic Control

Controlling the select through programmatic methods is possible by calling the select component instance directly. Note that this can be done through ViewChild as well.

Custom View Value

The value which appears inside the trigger can be customized easily through the viewValue input.

Nested Options

Instances of the fd-option component will be found regardless of how deep in the hierarchy they are. It also includes having options inside options, for instance. This feature allows for some very flexible usage. The example below showcases how to add a scrolling container around the options. Note that accessibility generally requires some text-content to be present in the actual option element.

Custom Trigger

The default trigger can be replaced by providing a template reference to the appropriate input. Note that the top-level element must be tab-indexed for the keyboard support to function correctly.

Adding and Removing Options

Adding and removing options dynamically is supported seamlessly, with very little extra work required from the developer. By design, if the user removes the option that is currently selected, the value will become undefined. This was implemented to prevent a user from keeping the active value of a previously removed option.

Form Usage

The select component can be used inside Angular reactive forms.
