Standard Calendar

This example also features blocked and disabled days.

You can use two-way data binding or simply ngModel to select a date or range of dates on a calendar. Calendars internally utilize an interface called CalendarDay, which contains the property date. To select a date you will create a new object that contains the property date, whose value is a FdDate model object. Then, using two-way binding syntax, set the selectedDay value to your new object. Note that selecting a day (or range of days) in this manner will override any block/disable functions. The disableFunction and blockFunction inputs can be changed and the calendar will automatically update (see example).

A disabled date or range of dates cannot be selected, usually applied to a large range of past or future dates. A blocked date or range of dates cannot be selected, usually applied to blackout or booked dates mixed in with available dates.

Range Selection Calendar

Allows the selection of a range of dates. Only the first and last days of the range are passed back.

The disableRangeStartFunction, disableRangeEndFunction, blockRangeStartFunction and blockRangeEndFunction inputs can be used in addition to the disableFunction and blockFunction inputs to set independent rules for the start and end dates. In this example, the user can only select a start date on or before October 10, 2019, and an end date on or after October 20, 2019.

Programmatic date change

Makes it possible to programmatically change the value of the currently selected date. The selectedDay property stores the date that the calendar highlights as the one that is currently selected. By programmatically changing the value of this property, the date that is highlighted on the calendar will change accordingly.

Monday Start Calendar

Use the startingDayOfWeek input to change the order in which the days of the week appear. 1 is for Sunday, 2 is Monday, 3 is Tuesday, etc.

Internationalization

The calendar uses a custom service called CalendarI18n to populate the month names, week days, etc. It also uses a separate service called CalendarI18nLabels to populate aria labels in the component itself.
