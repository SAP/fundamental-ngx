Basic example

Any element can be set as disabled using the [fdDisabled] directive. When passed as true or as an attribute, the [fdDisabled] directive will add .is-disabled, [disabled] and [aria-disabled] to the target element.

[fdDisabled] is independent directive and parent or child directives do not have any impact on it. Everything is quite opposite.

FnDisabledProvider

There are cases when you need to tap into the process of declaring certain elements disabled. For example, if you want to add the class is_very_disabled alongside default behavior when disabled and remove it when not. You can achieve that in two ways, one recommended and one side effect of having a notifier. We will go through the recommended way below.

Things to note:

  • FnDisabledProvider will also detect native HTML [disabled] attribute and will act upon it's change.
  • FnDisabledProvider will detect if any ancestor has [fdDisabled]="true" and will also set itself as disabled
  • When you add your recipient component as FnDisabledViewModifier it will be called immediately with current state and also after every consequent state change. You can add as many FnDisabledViewModifier as you want, they will be called one after another, in the order they have been added.
  • FnDisabledProvider ensures that you'll get correct information about current state of disability, taking into account changes related to self(both, invoked by the [fdDisabled] and as well as by adding native attributes to the host element) and the parent.
