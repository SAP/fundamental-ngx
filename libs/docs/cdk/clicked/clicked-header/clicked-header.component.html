Clicked event manager

During the development of accessible components, reoccurring behaviors are often found leading to frequent code duplication. One of these instances is interaction detection. Besides the "click" event, both "space" and "enter" can be an indication of the user's will to select something. This requires the developer to implement handlers for all three of these events. To eliminate that much-repeated code, utilize the fdClicked directive. The syntax is very simple, instead of writing (click)="onClickHandler($event)", you will write (fdkClicked)="onClickHandler($event)" and it will catch both "keydown.enter" and "keydown.space" events alongside the regular "click" event.

(fdkClicked) also works with the @HostListener decorator and with Renderer2.listen

Include ClickedBehaviorModule.forRoot() in imports array of your Root module to have access to the event handler.
