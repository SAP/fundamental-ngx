Basic example

The tool header component has three modes:

  • Desktop - the default mode, which is displayed on desktop devices. Content density should be Compact
  • Tablet - displayed on mid-sized devices. Content density should be Cozy
  • Mobile - displayed on mobile devices. Content density should be Cozy

Tool header component must have a productName and Logo. By default, the logo will be SAP logo, but you can override it by providing your own logo.

Auto mode example

The tool header component modes and orientations can be set to auto, which will automatically change the mode based on the element's width.

Note: [fdbToolHeaderAutoMode] will also update the Content Density of the tool header component Menu button example

The tool header component can have menu button, which will be displayed before the logo. It can be controlled with keyboard. In RTL, left arrow will output collapse event and right arrow will output expand event. In LTR it is the opposite.

Product switch example

The tool header component can have a product switcher

 Custom Logo example

By default, the logo will be SAP logo, but you can override it by providing your own logo, with your own attributes and actions

Tool Layout example

Tool header component is meant to be used with fdb-tool-layout component. It should occupy the top-section of the tool layout component.
