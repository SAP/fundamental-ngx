Basic example

Vertical navigation can accept multiple states, such as: expanded, snapped, popup.

Developers can set the state by using [state] input property.

Additionally, developers can define the visual styling for a particular device: desktop, tablet, mobile.

This can be set with [mode] input property which is desktop by default.

 Parent Navigation as Link (Two-Click Area)

By default, the Parent Navigation Item functions as a Navigation Group.

A small arrow icon shows the item's expanded/collapsed state in this scenario. In the snapped variant, clicking on such a navigation item will open a popover that displays the title and Child Items.

Although not recommended, the Navigation Item can function for navigation and expansion/collapse. In this case, Navigation component will automatically detect that external toggle arrow is needed and will render it. Clicking on the toggle arrow will show/hide Child Items, and clicking on the item will perform the navigation.

Selection Mode

There are 2 selection modes: router and click.

  • Router-based mode: Selection is determined by router link activation (default behavior)
  • Click-based mode: Selection based on item type:
    • Leaf items (e.g., "Home", "All Leads"): Clickable and selectable
    • Group headers (e.g., "Sales", "Reports and Analytics"): Only expand/collapse, not selectable
    • Parent items without linksOnly expand/collapse, not selectable
    • Two-zone items with links Text area is selectable, expander button expands/collapses
    • Disabled items: Not interactive in either mode

ID-based Selection:

  • Programmatic Selection: Use setSelectedItemById(id) to select items by their ID
  • Getting Items: Use getNavigationItemById(id) to retrieve specific navigation items
Embedded Mode

The Side Navigation must be placed underneath the Shell Bar. It must support the collapsed and expanded mode, which can be toggled via the side navigation menu located in the Shell Bar. For expanded mode set the state property of the navigation component to expanded, and for collapsed state set it to snapped.

On small screens (Size S), Side Navigation can open as a full-width overlay over the content area beneath Shell Bar. To enable this mode set isOverlay property to true. It will work only for expanded state.

Overlay Mode

The Side Navigation can be implemented in an Overlay Mode using Popover component, which is triggered by the side navigation button in the Shell Bar. On small screen sizes (600px) it should follow the behaviour of the Responsive Popover.
