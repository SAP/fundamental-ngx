Page Scrolling

In order to get table page scrolling use the next options:

  • Add [pageScrolling]="true" to trigger a fetch once the table is scrolled to the bottom.
  • Add [bodyHeight]="tableBodyHeight" to restrict a table body height so the vertical scrollbar can appear.
  • Add [pageSize]="itemsPerPage" to specify how many items there are per page.
  • Add [pageScrollingThreshold]="thresholdInPx" to indicate the threshold between the height of the remaining content and the need to load new data.

Consider that columns width automatically calculates only after the first set of data comes to the component.

 Virtual Scroll

When dealing with large amounts of data, it is recommended to use virtual scrolling to improve performance. To enable Virtual Scroll, use the following properties:

  • [virtualScroll]="true" - enables virtual scroll
  • [bodyHeight]="value" - sets the height of the table body (viewport)
  • [rowHeight]="value" - sets the minimum height of the table row if it is different from default for the current content density

Additionally, you may set the renderAhead input property to specify how many rows should be rendered ahead of the current viewport. Default is 20. Or control the table state on the application level to control for example a scrolling position so it can be properly restored.

Note that all rows must be of the same height in order for virtual scrolling to perform well. As mentioned before, you can set the minimum height with the [rowHeight] input. When using popping columns on smaller width screens, it is recommended to provide a pixel value to the [secondaryRowHeight] input. This input works similarly to [rowHeight] but instead applies to the popping column data that is placed beneath the original row.
It is imperative that cells in the popping columns are all the same height as well. If the data loaded in to the cells causes their contents to exceed the height of [rowHeight] or [secondaryRowHeight], and therefore causing the heights of rows to differ, we recommend using *fdpEditableCellDef or *fdpCellDef to define custom cell templates that will make the contents of the cell fit more predictably.

Outer Scroll

Add outerScroll to the fdp-table component for scrolling option outside of the table with sticky toolbar and table header.
