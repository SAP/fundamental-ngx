Handling large amount of data

One of the scenarios of using Platform Table component, is to render the large amount of items.

Storing this amount directly in memory may lead to a browser performance issues.

In order to keep the performance on a normal level, Platform Table component supports lazy loading of the items on scroll, and reusing rendered row components for row items.

Additionally, Platform Table component supports lazy loading for a child row items.

For this, developers need to pass Data source class that retrieves row's children into [childDataSource] input property.

IMPORTANT: due to complexity of the children lazy loading logic, [childDataSource] can only accept classes derived from TableChildrenDataProvider.

 Handling large amount of data and columns

Another scenario of using Platform Table component, is to render the large amount of columns.

Virtual scroll and lazy loading only handles vertical scrolling.

In order to keep the performance on a normal level, Platform Table component introduces ways to handle rendering in viewport which reduces DOM elements and improves performance, impacting horizontal scrolling.

For this, developers need to pass boolean into [onlyRenderVisibleCells] input property. Placeholder [fd-busy-indicator] for cell can be enabled by passing boolean into [useCellPlaceholder] input property
