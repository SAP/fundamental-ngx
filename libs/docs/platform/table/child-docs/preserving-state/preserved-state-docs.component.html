Preserving table state

In some cases, application should preserve the state of the table (current page, filters, grouping, sorting, etc) while navigating over pages.

To do so, developers can take advantage of using initial[Property] input properties and [property]Change output events.

Developers can also listen to a table body scroll event with the help of (tableScrolled) output event, when [pageScrolling] input property is true.

The example below demonstrates how to pass initial state and store changed state locally, then recreate the table component and pass updated state to it.

The logic behind it is to keep loaded data source items in memory, manipulate their properties when some event from table emits, and reuse this data on new table initialization.
