You can use [column] and [rank] attribute Change of column number for a field To change the column number in form-group-container for a field while screen size changes, property [columnLayout] can be used. it takes object in the form of XL: 3, L: 2, M: 1, S: 1. Change of isInline value for a field To change the isInline value in form-group-container for a field while screen size changes, property [isInlineLayout] can be used. it takes object in the form of XL: true, L: false, M: true, S: false. Here it shows that on L size screen, field should be dispalyed in vertical mode and when scren size changes to XL size then the field will be displayed inline(defaults to L layout value when value for XL not provided).

By default, form field layout contains the following grid definition:

  • Form field label: fd-col-xl--4 fd-col-lg--4 fd-col-md--4 fd-col-sm--12
  • Form field control: fd-col-xl--8 fd-col-lg--8 fd-col-md--8 fd-col-sm--12
  • Form field gap: fd-col-sm--0 fd-col-md--0 fd-col-lg--0 fd-col-xl--0

This layout can be easily changed with [labelColumnLayout], [fieldColumnLayout] and [gapColumnLayout] input properties.

These properties are available on multiple levels for more flexible layout configuration:

  • fdp-form-group
  • fdp-form-field-group
  • fdp-form-field

If fdp-form-field component is provided with input properties, they will be used as a grid columns definitions. Otherwhise these definitions will be inherited from fdp-form-field-group component. If form field is not wrapped into fdp-form-field-group, defintions will be taken from fdp-form-group
