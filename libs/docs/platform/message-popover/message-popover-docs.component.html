Message Popover with Simple forms

Message popover component supports both template-driven and reactive forms.

To define human-readable form item label text, developers need to use fdpMessagePopoverFormItem directive.

IMPORTANT!: Message Popover components bounds to the NgForm content children, and renders errors when ngSubmit event is fired.

Additionally, Message Popover Component supports asynchronous validators by waiting for form status to change from pending to other.

Message Popover with Form Container component

Message popover can also work with Form Container Component.

Syntax is similar with standard usage with some additional features.

Form container component also gives more advanced way of defining error texts and their types.

By placing *fdpFormFieldError directive with error parameters, developers can define own heading, description and error type.

Message Popover with custom error configuration

Message Popover component comes with predefined common error descriptions.

Developers can change these text by using withConfig method of PlatformMessagePopoverModule while importing it into application's module.

Error record could a simple string, or an advanced object with heading, description and error type.

If string is passed, error type will be automatically threatened as error.

 Message Popover with Form Generator component

Message Popover component can also work with components that have inner forms.

For this use-case developers need to pass forms and form fields as corresponding @input properties: forms and formFields.

As an example, Form Generator component which contains inner form that are not accessible with @ContentChildren for Message Popover.
