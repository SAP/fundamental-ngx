Variant Management Standard Usage

Variant management consists of two main components:

  • Wrapper: fdp-variant-management-wrapper
  • Renderer: fdp-variant-management

Wrapper component listens to the preset changes in components, thus, it should wrap all PresetManagedComponent components.

Renderer component can be placed anywhere on the page, and the instance of it should be passed back to the Wrapper component.

For the cases when Renderer component is placed inside Wrapper component and reachable with @ContentChild decorator, Wrapper will automatically find it, so users don't need to pass it instance directly.

Developers can define custom Author name via [userName] input property.

Variant object contains of data property, which should contain key-value object, where key is the preset-managed component name, and the value is actual preset configuration.

In case variant management is used with multiple identical components, component's name can be set individually via [name] Input property.

Variant Management with Table

Variant Management component supports configuring all table properties such as: search, filters, columns, pagination, grouping, sorting.

Variant Management with Dynamic Page

Variant Management component can also be used inside Dynamic Page component.

Please note: if you have multiple controlled components inside Dynamic page in different sections (header, content, etc.), you would need to wrap whole dynamic page in order for all components to be recognized.
