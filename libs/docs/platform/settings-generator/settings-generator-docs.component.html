Settings Generator Standard Usage

By default, Settings Generator renders with Sidebar layout.

Sidebar list item can accept thumbnail property which can be either Icon component configuration, or Avatar configuration.

Content section can be configured in two ways:

  • Form Generator component, which can generate various types of forms and it's controls;
  • TemplateRef, to render custom content.
 Displaying validation errors with Settings Generator

For better user experience, each form control type of Settings generator can have own validation rules (see Form Generator).

Since Settings generator is a component with complex layout and ability to display only active section, it is preferred to utilize Message Popover component with it.

Settings Generator automatically integrates with Message Popover and listens to Message Popover entry click to open appropriate section and group.

Settings Generator with Custom Layout

Developers can change default Sidebar layout to custom one.

To do so, you would need to generate a component that extends from BaseSettingsGeneratorLayout.

This abstract class contains of focusElementByPath abstract method, which is required for a cases when Settings Generator is used with Message Popover.

The purpose of this method is to find appropriate section and set it as active.

After component has been created, register it with SettingsGeneratorLayoutAccessorService.

In order to support Typings of your layout properties, you would need to pass a generic type of your Component's properties interface into SettingsModel.

Settings Generator with Custom Controls

Settings Generator uses Form Generator component inside to render forms.

Developers can extend default set of controls by providing additional control components via FormGeneratorService.
