Text type tabs

The text-only variant is one of the most common types. It allows longer labels, and can also display counters next to the text to indicate the number of items on the tab page.
To use the text type of the tabs you must pass input [type]="'text'"

Template-based Tab Content

You can define tab content using templates with IDs when using the data-driven approach with tabsConfig. This provides a clean way to connect tab configurations with template content.

Use the fdpIconTabBarContentTemplate directive to mark templates with unique IDs, then reference these IDs in your tab configuration using the contentTemplateId property.

Text type tabs (column mode)

To use this mode you need to pass input[layoutMode]="'column'"

 Icon type tabs Icon tabs are also common tab types. These round tabs can be populated with any icon from the SAP icon font. Labels are optional. If you decide to use labels, use them for all tabs. You can use counters as needed.
To use the icon type of the tabs without labels you must pass input [type]="'icon-only'" Icon type tabs with labels To use the icon type of the tabs you must pass input [type]="'icon'" Filter type tabs If you build the tab bar as a filter, it can comprise two parts:
  • An “all” tab on the left (optional) This tab shows the total number of items, and describes the type of item (for example, 189 Products). To use it pass [showTabAll]="true"
  • Tabs for specific filters Use the tab text to indicate the filter attribute. We strongly recommend showing a counter on every tab.
To use the filter type of the tabs you must pass input [type]="'filter'" Process type tabs You can also use the tab bar to depict a process. In this case, each tab stands for one step. To use the process type of the tabs you must pass input [type]="'process'" Nested tabs

Nesting allows deeper hierarchies with indentations to indicate the level of each tab.
Nested tabs will be displayed automatically if the tabConfig has the field subItems

This feature is available only for tabs with [type]="'text'" Reordering

You can allow users to rearrange the tab order in a desktop environment (property: enableTabReordering). If this feature is enabled, users can drag and drop tabs to reorder them, either directly on the tab bar or inside the overflow menu. It is also possible to drag and drop tabs from the tab bar to the overflow menu and vice versa.

This feature is available only for tabs with [type]="'text'" Overflowing

When the screen space does not allow to show all tabs on the main tab bar, an overflow appears on the far right, containing all remaining tabs that do not fit on the screen.

Configurable paddings

You can set horizontal paddings by adding a specifying the size of the paddings. Available values: 'sm' | 'md' | 'lg' | 'xl' | 'xxl' | 'responsive-paddings'

Closable tabs

Icon tab bar items can be made closable.

In order to make item closable, pass closable: true property of the item.

 Tabs content

In addition to data-driven approach, Icon Tab Bar can render content tabs with projected fdp-icon-tab-bar-tab components.

Async Tabs

The Icon Tab Bar supports asynchronous tabs with dynamic labels, counters. Additionally, it can render content tabs using projected fdp-icon-tab-bar-tab components, following a data-driven approach.

Multi Click Area

In case of two click areas for tabs with sub-tabs, the behaviour for regular tabs (without sub-items) remains unchanged, like described above the tabs get selected immediately.

Tabs with two click areas behave differently: when the main part of the tab is clicked, it is highlighted and then selected immediately.

Clicking the second area opens a menu (Popover) containing the sub-items. Once a sub-item is clicked, the main tab is highlighted as selected.
