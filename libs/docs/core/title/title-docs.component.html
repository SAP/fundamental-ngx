Semantic Levels Headings have 6 semantic levels (H1-H6). Use only one H1 per page and maintain ascending order without skipping levels (e.g., H1 → H2 → H3). The component automatically detects the semantic level from the element tag. Visual Styling Use [headerSize] to style headings differently from their semantic level, maintaining proper document structure while achieving design requirements. Text Truncation By default, title text truncates with ellipsis when it exceeds its container width. Text Wrapping Use [wrap]="true" to wrap overflowing text instead of truncating. Default Size via Dependency Injection Parent components can provide a default header size for all nested titles using the DEFAULT_TITLE_SIZE injection token. This is useful for containers like dialogs that want consistent title styling without manually setting [headerSize] on every title.

Priority order:

  1. Explicit [headerSize] input (highest priority)
  2. Injected DEFAULT_TITLE_SIZE from parent
  3. Auto-detected from HTML tag name (h1-h6)

Example usage:

import { DEFAULT_TITLE_SIZE } from '@fundamental-ngx/core/title';

@Component({
  providers: [{ provide: DEFAULT_TITLE_SIZE, useValue: 5 }]
})
export class MyDialogComponent {
  // All fd-title elements inside will default to size 5
}
