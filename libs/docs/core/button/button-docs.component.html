Button Types

The button component provides seven distinct visual styles to convey different semantic meanings and importance levels. Use the [fdType] input to set the button type.

  • Standard (default): Primary actions without special semantic meaning. No fdType needed.
  • Emphasized: The main call-to-action on a page, draws maximum attention.
  • Ghost: Secondary actions with minimal visual weight, shows border on hover.
  • Transparent: Tertiary actions with the least visual prominence.
  • Positive: Successful or affirmative actions (e.g., approve, confirm).
  • Negative: Destructive or rejecting actions (e.g., delete, cancel).
  • Attention: Warning actions that require user attention.

Note: Positive, Negative, and Attention buttons automatically generate accessible aria-label and aria-description attributes based on the button type for screen readers.

Menu Button

Menu buttons indicate that clicking will open a dropdown menu or popover. Setting [fdMenu]="true" automatically adds a dropdown arrow icon to the right side of the button.

Menu styling works with all button types and can be combined with icons via the [glyph] input. This is commonly used for:

  • Dropdown menus and action lists
  • Split buttons with multiple options
  • Filter and sort controls
 Button Sizes

Buttons support two density modes to adapt to different screen sizes and use cases. Use the content density directives to control button size:

  • [fdCozy]: Default size, optimized for desktop and touch interfaces (recommended for most use cases)
  • [fdCompact]: Reduced vertical padding for dense layouts like data tables or toolbars

Content density can also be set globally or inherited from parent containers using the fdContentDensity directive.

Button with Icons

Buttons can display icons to provide visual context for actions. Use the [glyph] input with an icon name from the SAP icon font to add an icon.

Icon Inputs:

  • [glyph]: Icon name (e.g., "cart", "filter", "accept")
  • [glyphPosition]: Icon placement - "before" (default) or "after" the label
  • [glyphFont]: Custom icon font family (defaults to SAP icons)

Icon Patterns:

  • Icon + Label: Best for clarity, combines visual and text cues
  • Icon Only: Space-saving option for toolbars. Always provide [ariaLabel] for accessibility

Accessibility Note: Icon-only buttons require an ariaLabel for screen readers.

 Button State

Buttons support multiple state inputs for different interaction patterns and accessibility needs:

Disabled States:

  • [disabled]: Native HTML disabled attribute. Prevents all interaction and is not focusable. Best for temporarily unavailable actions.
  • [aria-disabled]: Indicates disabled state to screen readers while keeping the button focusable and interactive. Useful for showing tooltips explaining why an action is disabled.

Learn more about disabled vs aria-disabled: disabled attribute | aria-disabled

Toggled State:

  • [toggled]: Visual and semantic toggled state. Sets aria-pressed="true" for screen readers. Use for toggle buttons that maintain on/off state (e.g., bold/italic formatting, filter activation).

Other States:

  • [selected]: Indicates selection in button groups or toolbars. Sets aria-selected="true" for accessibility.
  • [type]: Native button type attribute. Defaults to "button" to prevent form submission. Set to "submit" or "reset" for form buttons.
Toggled Button

Toggle buttons switch between two states (on/off) to control features or settings. The [toggled] input controls the visual pressed state and sets aria-pressed for accessibility.

When to Use Toggle Buttons:

  • Text formatting controls (bold, italic, underline)
  • View mode switches (list view vs. grid view)
  • Filter activation/deactivation
  • Feature toggles that persist during a session

Implementation:

Bind the [toggled] input to a boolean property and update it on click events. The button automatically applies the toggled visual style and ARIA semantics.

Note: Toggle buttons work with all button types. Use emphasized, positive, or attention types to indicate the importance or impact of the toggled state.

Badge Buttons

Badges display notification counts or status indicators on buttons. They're commonly used for shopping carts, notification counters, or collecting items across multiple pages before triggering an action.

Usage:

Project the <fd-button-badge> directive inside a button and use the [content] input to set the badge value (number or short text).

Supported Button Types:

Badges work with: Emphasized, Standard, Ghost, and Transparent button types.

Constraint: Badge content must not exceed 4 characters to maintain visual clarity and avoid layout issues. Playground Area
