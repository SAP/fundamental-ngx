Generic Tile Sizes Tiles are responsive and adapt to different screen sizes. The default large tiles are optimized for screens larger than 374px. For smaller screens, use size="s" to display compact tiles that fit mobile viewports better.

Tiles support two width dimensions:
  • 1×1 (default): Standard single-width tile
  • 2×1: Double-width tile using [double]="true" - useful for displaying more content or creating visual hierarchy
Generic Tile Content Columns Tile content areas (header, body, and footer) can be organized into two columns for better content layout. Use the [twoColumn]="true" input on fd-tile-header-content, fd-tile-content-text, or fd-tile-footer-text directives to split content into two columns with 0.25rem vertical spacing. This layout is particularly useful for displaying label-value pairs or organizing related information side-by-side.
Launch Tile Launch tiles are used as shortcuts to applications or specific features within your application. They typically display an icon, title, and subtitle to help users quickly identify and access functionality. Use type="launch" to apply the launch tile styling.

Usage as Link: Launch tiles often navigate to other pages. Use the attribute selector on a native link for optimal behavior: <a fd-tile type="launch" href="/dashboard">. This provides native browser navigation features including right-click context menu, keyboard shortcuts, and proper link semantics.
KPI Tile KPI (Key Performance Indicator) tiles display important metrics and performance data in a prominent, easy-to-scan format. Use type="kpi" to create KPI tiles. These tiles typically combine numeric content directives to show values, trends, and comparisons.

KPI tiles leverage the fd-numeric-content directive family to display:
  • Primary KPI values with units
  • Scale indicators showing change (with positive, negative, critical, or informative states)
  • Trend arrows indicating direction
  • Launch icons for navigation to detailed views
Action Mode, Badges, Feed Tiles, and Interactivity Action Mode: Use [action]="true" to place a tile in action/edit mode. This adds an overlay to the tile with action controls. Provide fd-tile-action-close and fd-tile-action-indicator directive-enhanced buttons to handle edit and close actions. Action mode is useful for allowing users to reorder, remove, or configure tiles in a dashboard.

Badges: Badges display brief, important information like "New" status. Use badges to highlight tiles that have been recently added or updated. Note that badges are not displayed when tiles are in action mode.

Feed Tiles: Feed tiles (type="feed") are designed for notification streams and activity feeds. They display the most recent notification when no new updates are available, and highlight new content when present.

Clickable Tiles: Control tile interactivity with the [clickable] input. When [clickable]="true", tiles become keyboard-focusable (tabindex="0") and emit (tileClick) events on click, Enter, and Space key presses. This is only applicable when using <fd-tile> as a custom element. For link behavior, use <a fd-tile href="..."> instead.

Accessibility: Enhance tile accessibility by setting id, ariaLabel, ariaLabelledby, and role inputs as needed. These properties help screen reader users understand tile purpose and content.
