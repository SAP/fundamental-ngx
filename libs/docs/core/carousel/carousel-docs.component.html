Carousel with one active item By default fd-carousel displays one active item and others remains hidden. Pagination container appears on bottom with dot indicators with active dot to show active item.

Notes:
By Default images used as carousel item will drag on non-touch devices, instead of swiping. To achieve swiping behavior for these images, below solutions can be applied.
  • use css solution. pointer-events: none
  • use javascript solution. [draggable]="false" ondragstart="return false;"
 Carousel with vertical direction carousel movement is possible in vertical direction as well. Carousel with multiple active item Carousel can display multiple active item at a time as well. On left navigation button click, right most item hides and a new item is shown on left side. On right navigation button click, left most item hides and a new item is shown on right side. visibleSlidesCount property is used to set, how many item will be active at a time. Ex. [visibleSlidesCount]="3". Carousel with dynamic items Carousel items can change dynamically, eg: when used inside search page. Carousel will reset when new items are added on Carousel. Carousel with no page indicator Page indicator is optional in carousel. It has option to hide page indicator by setting pageIndicator=false. Carousel with hidden navigation buttons In Mobile display, navigation buttons are not required for carousel. Navigation will be done by swiping the active item, either left side or right side. Carousel with navigation inside content area Navigation buttons can be placed in page indicator as well as inside content area.By default navigation button is placed inside page indicator container. For placing navigation button inside content area use navigatorInPageIndicator=false. Carousel with looped navigation Carousel item navigation can be looped by setting loop=true. By default loop is false and looping is not enabled on carousel. Carousel error message when no item is loaded. Use Message Page component to create the error message. Error message can be displayed as carousel content when loading item is not successful. Carousel item loading indicator When loading of Carousel content takes time, then for better user experience, busy indicator can be shown. Automatic slides count

It is possible to set automatic slides count for the carousel by providing visibleSlidesCount="auto" input property of the fd-carousel component.

in this case carousel will automatically calculate amount of fitting slides of the current carousel's viewport.

Background modifiers

The background of the Carousel Content and Page Indicator Container can be changed by providing [contentBackground] and [pageIndicatorBackground] input properties.

Background options can be following:

  • translucent
  • transparent
  • solid

The default background for the Carousel Content is translucent.

The default background for the Page Indicator Container is solid.

Additionally, developers can pass [noPaginationContainerBorder]="true" to remove the border top or bottom of the Pagination container.
