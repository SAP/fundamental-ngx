User Menu

The User Menu is an integral part of the Shell Bar, accessible by clicking the avatar located at its far-right end. It provides user-specific options and remains available across all screens. In addition to standard options, the User Menu can include product-specific functionality.

The User Menu can display the following:

  • Current user information
  • Switch between accounts
  • User Menu items

When to use

  • To present user-specific information, settings, and actions.
  • To enable account switching.
  • To display product-related and legal information.
  • To facilitate Sign In or Sign Out actions.

When not to use

  • Navigation: Avoid including navigational elements.
  • Home Page Customization: Exclude items related to customizing the home page.
  • Inactive Items: Hide inactive items instead of displaying them as disabled.
  • Content Overload: Keep the menu concise by limiting links to essential items only.
  • Dynamic Content: Avoid altering the menu content within the product.

Anatomy

Components

  • fd-user-menu
    • mobile input to contol the mode (default/mobile)
  • fd-user-menu-control
    • clicked output
  • fd-user-menu-body
  • fd-user-menu-content-container
  • fd-user-menu-list
  • fd-user-menu-list-item
    • uniqueId input (string) - to set the id of the list item. A default one is provided.
    • icon input (string) - icon (glyph) name for the menu list item (optional)
    • text input (string) - text for the menu list item (required)
    • textId input (string) - id for the text element. A default one is provided.
    • hasSubmenu input (boolean) - whether the item has a submenu
    • submenu input (TemplateRef) - a template ref for user menu list item submenu
    • selected input (boolean) - whether the item is selected
  • fd-user-menu-sublist
    • hasIcons input (boolean) - whether the sublist items have icons
  • fd-user-menu-footer

Directives

  • fd-user-menu-header
  • fd-user-menu-header-container
  • fd-user-menu-user-name
    • truncate input (boolean) - whether the text should truncate with ellipsis after the second line.
  • fd-user-menu-subline
    • truncate input (boolean) - whether the text should truncate with ellipsis.
