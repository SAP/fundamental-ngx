Step Input Step Input can be used in default and compact size modes. Use fdCompact to enable compact mode. Custom configuration Step Input offers multiple configuration options like:
  • Custom step value
  • Custom button icons
  • Input text alignment
  • Optional format grouping
  • Setting upper and lower boundary limits for the value
  • Specifying minimal or maximal number of fractional digits
 Custom locale By default Step Input uses users locale. To directly set locale used within the Step Input, use [locale] input. Input states Step Input control supports semantic states. Use [state]="success | error | warning | information" input to set one of the semantic states. Step Input label Step Input label is displayed next to the step control and should be used to describe the unit of the value selected with the Step Input control. Currency Step Input supports currency mode, automatically formatting the currency and displaying currency sign in Step Input label. To use currency mode set mode="currency" and currency="USD". To get more familiar with the options of customizing how currency can be formatted check Intl.NumberFormat documentation.
If currency is used together with custom [unit] input, unit is going to override currency symbol in label. Reactive and Template Driven forms Step Input can be used both with Reactive and Template Driven forms.
