Simple Menu

When to use: Use the menu component to display a list of actions or options that appear on demand. Menus are ideal for context menus, action lists, and navigation dropdowns where space is limited.

To create a simple menu, use the following directives:

  • fd-menu-item - wraps each list item
  • fd-menu-interactive - makes the content interactive (button or anchor)
  • fd-menu-title - displays the menu item text

Attach the menu to any HTML element using the fdMenuTrigger directive.

 Menu with Icons

When to use: Add icons to menu items to provide visual cues that help users quickly identify actions. Icons are especially useful for frequently used actions or when the menu contains many similar options.

Menu items can be enhanced with additional visual elements:

  • fd-menu-addon - adds an icon to the menu item. Use position="before" to place the icon before the title
  • fd-menu-shortcut - displays keyboard shortcut information for power users
 Menu with Separator

When to use: Use separators to group related menu items together. This improves scannability when menus contain many items or distinct action categories.

Add a visual separation line between menu items using the [hasSeparator] input on the fd-menu-item component.

Menu with Scrollbar

When to use: When menus contain many items that exceed the viewport height, the component automatically handles overflow with a scrollbar using fd-scrollbar.

This ensures users can access all options while maintaining a consistent viewport size. No additional configuration is required.

Menu with Submenu

When to use: Use submenus to organize hierarchical navigation or categorize actions into logical groups. Submenus help reduce clutter in the main menu while keeping all options accessible.

To create nested menus:

  • Create the main menu structure
  • Define submenus using the fd-submenu component
  • Connect submenus to parent items using the [submenu] input
  • Add fd-menu-addon with [submenuIndicator]="true" to show a visual indicator for items with children

Use (activePath) to track the currently hovered/selected item path. Customize hover expansion timing with [openOnHoverTime].

Extended Menu Items

When to use: Use extended menu items when you need interactive controls within the menu, such as toggles for quick settings, segmented buttons for mutually exclusive options, or input fields for filtering.

Toggle buttons: Use button[fd-menu-interactive][toggle] for on/off settings. Supports template-driven and reactive forms via ControlValueAccessor.

Segmented buttons: Use li[fd-menu-item][fdMenuSegmentedButtonHeader] as the group container with [(ngModel)] binding. Each option uses li[fd-menu-item][fdMenuSegmentedButtonOption][value]. Provide a custom valueComparator function if you need non-strict equality comparison.

Input fields: Use input[fd-form-control][fd-menu-item-input] inside div[fd-menu-interactive] for inline text input.

Mobile Mode

When to use: Enable mobile mode for touch-friendly interfaces where the menu should display as a full-screen dialog. This provides better usability on small screens and touch devices.

To enable mobile mode:

  • Set [mobile]="true" on the menu component
  • Optionally configure appearance with [mobileConfig] using MobileModeConfig

Mobile mode can be configured globally using the MOBILE_MODE_CONFIG token. See the Global Configuration page for details.

Programmatic Control

When to use: Use programmatic control when you need to open, close, or toggle the menu based on application logic, or when building custom keyboard shortcuts or accessibility features.

The menu component provides several ways to control it programmatically:

  • [(isOpen)] - two-way signal binding for reactive state management
  • open(), close(), toggle() - imperative control methods
  • [config] - pass a PopoverConfig object for consolidated configuration

Available events:

  • (beforeOpen) - fires before the menu opens, useful for lazy loading content
  • (isOpenChange) - fires when the open state changes
  • (activePath) - fires when the active menu item path changes (for submenus)
 Disabled States

When to use: Disable the entire menu when a triggering condition isn't met, or disable individual menu items when specific actions are unavailable based on the current application state.

Disabled menu: Set [disabled]="true" on the fd-menu component to prevent the menu from opening. The trigger button will appear disabled.

Disabled items: Set [disabled]="true" on individual fd-menu-item elements. Disabled items are visible but not interactive and are skipped during keyboard navigation.

Placement & Triggers

When to use: Customize menu positioning and trigger behavior to fit your UI layout and interaction patterns.

Placement: Use the [placement] input to control where the menu appears relative to the trigger. Options include bottom-start (default), top, left, right, and their -start/-end variants.

Fill mode: Use [fillControlMode] to match the menu width to the trigger: at-least sets a minimum width, equal matches exactly.

Triggers: Use the [triggers] input to control how the menu opens. The default is ['click']. For hover behavior, pass a TriggerConfig array.
