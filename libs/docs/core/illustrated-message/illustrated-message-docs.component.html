Large Large illustrated messages are ideal for full-page scenarios such as empty states, error pages, or initial landing screens. The illustration automatically scales responsively based on the container width.

The [svgConfig] object should define SVG configurations for all sizes (large, medium, small, and xsmall) to ensure proper responsive behavior as the container resizes. Medium Medium illustrated messages are designed for use in dialogs, message boxes, or page sections where space is more constrained than full-page scenarios. Set the type property to "medium" to explicitly use this size.

The [svgConfig] should include configurations for medium, small, and xsmall to support responsive scaling within the dialog or container. Small Small illustrated messages work well in medium-sized cards, sidebars, panels, or other compact UI areas where you need to communicate a state or message with limited space. Set type="small" to use this size explicitly.

The [svgConfig] should provide configurations for small and xsmall to handle responsive behavior as the container shrinks. Extra Small Extra small illustrated messages are suited for very compact UI elements like small cards, list items with illustrations, or narrow sidebars where space is highly constrained. Set type="xsmall" to explicitly use this minimal size.

At this size, only the xsmall configuration is required in [svgConfig] since the container won't scale to larger sizes. Inline SVG

In scenarios where external SVG URLs are not practical (such as offline applications, bundled assets, or when avoiding external HTTP requests), you can provide inline SVG content directly as a string.

Use the file property in your [svgConfig] instead of url and id. The component will render the inline SVG and automatically handle sizing and responsiveness. Include file configurations for all responsive sizes (large, medium, small, xsmall) for optimal responsive behavior.
