Interactive card with role listitem The listitem is the recommended role for card. With this role, all interactive variants of the card can be covered:
  • Non-interactive card
  • Non-interactive card with interactive elements inside
  • Interactive card
  • Interactive card with interactive elements

IMPORTANT: only the list item role allows nested interactions, which means: the card can be interactive itself while also having interactive elements inside. It is not recommended to use interactive cards outside list containers!
The card container MUST be a list fd-card-list, even when there's only one card. The container of the list should have role region and aria-label. If there's no container for the list, the label has to be placed on the card container list.
Non-interactive card Non-interactive cards have role="region" (default). In this case the card can be placed in any container. This role covers the interactive variants:
  • no interactivity
  • no interactivity with interactive elements inside (including interactive header)
 Header Interaction - non-interactive Cards The interactive header cannot be combined with the whole card being interactive. Typically the action triggered by an interactive header would be identical with the action of an interactive card. Which to use when is up to the LoB / product and their use cases and standards.

For interactive header set the main card header property[interactive]="true".

To the value of aria-description, aria-label and title for the main header, use the [title] property. Standard Card The Badge for Cards can contain Icon and Text, Text only or Icon only. There are no interaction states on Badge for Cards. Badge on Cards consumes the Inverted Object and its color variations. There can be max 2 badges per Card.
  • Use the badge property of card to set the text of the badge. Accepts a string value. For the second badge use secondBadge property.
  • The badgeIcon property of card is to set the icon. Accepts a string value, the glyph name of the icon. For the second badge use secondBadgeIcon property.
  • To set the color of the badge add an integer from 1 to 10 for the badgeColor property. For the second badge use secondBadgeColor property.
    • Accepts a number value.
  • To enable the second set of Accent colors set badgeColorSecondary to true. Accepts a boolean value. For the second badge use secondBadgeColorSecondary property.
  • For status colors use the badgeStatus property. Available values: 'negative' | 'critical' | 'positive' | 'informative' | 'neutral'. For the second badge use secondBadgeStatus property.
  • aria-label for the Card badge can be set through badgeAriaLabel property. Accepts a string value. For the second badge use secondBadgeAriaLabel property.

Object card The object card can display a single object or a group of objects. Calendar card The calendar card displays calendar within card. Quick view card Quick view cards are single-object cards. They display the basic details for one object, such as the name, address, and phone numbers for a contact. List card List cards display a set of items or links in a list format. Link List card Link list cards allow you to display a collection of links or images that can reference both internal and external targets.Links and images are handled as two separate variants: list and image. Media card The Media Block is flexible in height but fixed in width to the card size. The media can only contain an image / graphic OR solid color plus text. The Image can fill the media container or can come with a padding of 1rem (results in 16px) top, left and right. Corner radius 0.5rem (results in 8px). For this option set hasPadding to true.

A title on Media Block is possible if using a solid colored background an NO image! To ensure legibility and ACC standards in the Media Block / Banner, always make sure a sufficient contrast ratio is achieved.

Solid background colors can be bright Shell Category Colors or the lighter Legend Background Colors. Use the shellColor and legendColor properties of fd-card-media component to set the background.

shellColor can be a number from 1 to 16,
legendColor can be a number from 1 to 20. Media card To highlight certain content in a crisp and highly visual way, the Media Block offers the flexibility to be designed as a stand-alone banner. Add cardType="banner" to the fd-card to achieve this type of Card.

When the Card is used as a banner, the solid background can be applied on the whole media container or on the content container, in cases where it's used together with an image or as an overlay. The properties shellColor and legendColor can be set on both, fd-card-media and fd-card-media-content-container components. shellColor accepts a number from 1 to 16, legendColor can be a number from 1 to 20.

The content container can be displayed as an overlay in case of using an image / a graphic behind. For this functionality isOverlay property of fd-card-media-content-container should be set to true.

When there's no title or heading to describe the Card, the user can use the ariaLabel property of fd-card to set its aria-label.
