Card
The Card is a container that consists of a header, a content and a footer area. It takes the width of its parent. The header could be placed either above or below the content and is not separated with a line. The footer area is optional.

Cards can display different types of content, such as:
  • informative text
  • a chart (Analytical Card, Bar Chart List)
  • a list (List card, Link List card)
  • a table (Table Card)
  • a calendar (Calendar Card)
  • an object or group of objects (Object card, Quick view card)


Card sizes vary depending on the layout, and they are not editable. A card can focus on either a single object or topic, or on a group of objects. Several cards can reference the same application, but the information should be distinct from one card to another.



Card building blocks
  • card list container fd-card-list: interactive cards should have role="listitem" and be placed in a list, even when there's only one card. The fd-card-list directive will apply the necessary role and attributes to the container and is responsible for setting the ariaPosinset and ariaSetsize of each card.
  • fd-card is the Card container.
    • interactive when set to true adds interactive behaviour to the cards
    • isLoading set to true if the card is in loading state
    • cardType can set the Cart type, such as analytical, table, list, object, etc.
  • header (optional) fd-card-header: the card header is a flexible combination of 3 block areas: main header, numeric header and extended header. The header displays a mandatory title, indicating what the card is about and functions as a navigation control that directs the user to the parent app.
    • main header fd-card-main-header: the main card header block shows the basic information of the card. For interactive header add [interactive]="true"
      It can contain:
      • avatar (optional) fd-avatar
      • title fd-card-title
      • subtitle(optional) fd-card-subtitle
      • counter (optional) fd-card-counter
      • button (optional) fd-card-header-action
      • KPI header (optional) fd-card-kpi-header
    • extended header fd-card-extended-header: the extended header is a flexible block to hold various other components, like time stamp, rating, tags, label/value, long text, numeric values, etc. The components can be arranged in left column fd-card-header-column or right column fd-card-header-column [align]="'right'", each column is suggested to have maximum 3 lines (rows) fd-card-header-row for placing the components.
    • numeric header fd-card-numeric-header: the numeric header block is designed for displaying numeric information. It consumes Numeric Content (Horizon) and can show additional qualifying information and side indicators, if required. The numeric content is placed inside fd-card-numeric-container, and the indicators fd-card-indicator-title and fd-card-indicator-value are inside fd-card-indicator. An additional information, like a label, can be added in a second row fd-card-header-row
  • content fd-card-content: the content area is reserved for application content.
  • footer (optional) fd-card-footer: the footer displays a list of actions *fdCardFooterActionItem that can be performed on the card. When link is too long, or there is no more place for actions, overflow button should appear.
