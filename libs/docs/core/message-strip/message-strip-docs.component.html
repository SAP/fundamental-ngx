Message Strip Types A number of message strip types are available. They offer preset styling for different contextual situations. Can be one of 'warning', 'success', 'information', 'error' or normal (default). Message Strips Without Icons The default message strip has no icon. Information, Success, Warning and Error message strips by default are rendered with icons. To remove them set the [noIcon] property to true. Message Strips With Customizable Width The width of the message strip can be customized by the [width] and [minWidth] properties. Message Strip Alerts

Message Strip Alerts are used to display a short, important message in a prominent location on the page. They are used to inform users of a situation that requires their attention or action. They are not part of the Fiori Design guidelines.

This example shows usage of the MessageStripAlertService. You can set custom footer for rendering actions. You can use TemplateRef<{{ '{' }}$implicit: MessageStripAlertRef{{ '}' }}>, ordinary string or the component.

If you choose to have component as a content, you can use MessageStripAlertRef to close the alert, or get some data from it.

Message stripe alert containers are responsive. Below the M(1024px) breakpoint, the alerts from all the positions are unified into either bottom-middle or top-middle positions, depending on their vertical position requirement and they are ordered by the time of appearance.

If from the vertical positions(bottom or top) only one is used, then the container will take all the viewport height, and it will have a scrollbar if the content is bigger than the viewport. If both vertical positions are used, the container will take only half of the viewport height.

Auto Dismiss Message Strip Alerts

Message Strip can be automatically dismissed after a specified amount of time. To enable this feature, use AutoDismissMessageStripDirective directive.
