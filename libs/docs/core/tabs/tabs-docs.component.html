Simple Tabs Simple tabs can be built by setting specified properties to fd-tab. There is way to define own templates, or use it with [routerLink]. There is no compact mode on inline tabs. Form element inside tab content Adding form element inside tab content to show the keyboard accessibility navigation for tab component Tabs with Counter Tabs with counter can be built by setting specified properties to fd-tab. There is way to define own templates, or use it with [routerLink]. There is no compact mode on inline tabs with counter. Icon Only Mode The Icon Only mode can be used, by adding mode="icon-only". It requires different structure usage. Tabs in Icon Only mode can be built by setting specified properties to fd-tab. There is way to define own templates, or use it with [routerLink]. Process Mode The Process Mode can be used by adding mode="process". As on icons it has different structure than tab any other mode. It can be built by setting properties, defining own template, or using it with [routerLink]. Filter Mode The Filter Mode can be used by adding mode="filter". It has different structure than tab any other mode. It can be built by setting properties, defining own template, or using it with [routerLink]. To make filter Mode Tabs looks properly, it's mandatory to add [header]="true" property to first tab item, or in case of custom template use different structure, as it is shown in example. Programmatic Selection To programmatically open/close the tab, get tab reference and call .open(true | false) method. Adding and Removing Tabs The tab component is very flexible, and supports dynamically adding or removing tabs. Collapsible Overflow When tabs cannot fit in the navigation header, use [collapseOverflow]="true" to move overflowing content into collapsed tabs list. To set maximum number of tabs visible in the header, use [maxVisibleTabs]. Text visible in the expand collapsed tab can be customized using [expandOverflowText] property. Stacked Content In stacked content mode, all of the tabs will be initially expanded and the navigation between the tabs will be handled using with scroll. To enable stacked content set [stackContent]="true". Collapsible Tab By default expanded tab can be collapsed only by opening another tab. To allow user collapsing expanded tab without opening another tab use [collapsibleTabs]="true". Default Tab The first tab is the default tab. To specify default tab, use [defaultTab] input property and pass id of the tab. Also, on tabs array change the selected tab kept selected (if possible).
