Side Navigation Side Navigation is a vertical navigation component that is placed on the left side of the screen. It is used to navigate between different sections of the application. you need to add fd-side-nav component, and inside it, you need to add fd-side-nav-main and fd-side-nav-utility elements. Inside fd-side-nav-main you need to add fd-nested-list element, and inside it fd-nested-list-item elements. To have a header, use fd-nested-list-header element. To have a link, use fd-nested-list-link element. To have a selected link, use [selected]="true" on the fd-nested-list-link element. To have a nested list, use fd-nested-list element inside fd-nested-list-item element. To have a nested list header, use fd-nested-list-header element. Side Navigation With Multiple Levels and Selected Option. To have multiple levels of hierarchy in the left navigation, use fd-nested-list inside the fd-side-nav-main element. To have selected option, use [selected]="true" on the fd-nested-list-link element. Side Navigation with Programmatically State Change. Side Nav also allows to control the expanded and selected state, from outside. Side Navigation with Icons To put icons into the Side Navigation, there is no need to add [textOnly]="true" to fd-nested-list anymore. Side Navigation Condensed Mode For Mobiles, or lower screen devices there is condensed mode. It's recommended to use it with icons, because icons are the only elements, that are visible in first level navigation. Also nested lists should be placed in the fd-nested-list-popover element. Side Navigation Generated from Configuration Object Side Navigation component can be generated by configuration object SideNavigationModel, by passing options and content. Side Navigation Condensed Mode Generated from Configuration Object Side Navigation component generated by configuration object can be used along with condensed mode. The popover is added automatically.
