Simple Datetime Picker The simplest form of the datetime picker. It makes use of ngModel to bind to the desired value. The alternative is to use the date input combined with the dateChange output. There is also added [placement] attribute which allows us to decide where popup should be shown The value for the input only changes the model on blur or an enter/return keystroke. Some default options that belong to the time component such as [displaySeconds], [displayMinutes], [displayHours] and [meridian] are based on the DateTimeFormats.display.dateTimeInput options.
For example if Datetime picker display format option contains hours, minutes and day period info then default options will be the next: [displaySeconds]="false", [displayMinutes]="true", [displayHours]="true" and [meridian]="true". The listed above options are applied to the time component only (used internally) and can be defined explicitly if needed. Programmatic Change It is possible to programmatically change the value of the date of the datetime picker and still conserve full functionality. Null Validity Null input values are considered valid by default. Use [allowNull]="false" to make them invalid. Formatting Providing a custom format for the dates is possible through providing DATE_TIME_FORMATS config, this format applies to all the datetime-picker component. It is possible to override the custom format for any specific datetime-picker component through providing [customDateTimeFormat] input. Note that when providing a custom format, the [placeholder] input should be set to match the new format as well. The dateTimeFormat pipe can also be utilized to convert the date into a string according to the provided format. Additionally, support for the dayPeriod pattern has been added to handle day periods like morning, afternoon, evening, and night. To use these transformations in your templates, you can chain the dayPeriodFormat and translateDayPeriod pipes together. For example:
Selected: date | dayPeriodFormat: dateTimeFormatWithDayPeriod | translateDayPeriod
This example shows how to transform a selected date using the custom day period format, which can then be localized or further customized with the translateDayPeriod pipe. The calendar legend feature displays a visual guide explaining what each special day marker means. Special days are marked in the calendar with distinct colors and styles, while the legend below explains each marker type. Enable the legend by setting [showCalendarLegend]="true" and providing specialDaysRules to define which days should be marked and how they appear.

Focusing on legend items automatically highlights the corresponding days in the calendar, providing immediate visual feedback and enhanced accessibility. The legend is keyboard navigable using arrow keys and integrates seamlessly with the datetime picker's calendar view.

How to Use

Define your special days rules with a specialDayNumber (1-20) for the marker color, a rule function that returns true for dates to mark, and legendText describing the special day. Optionally set appointment to true for a circle marker. Customize the legend layout with [legendCol]="true" for a single column or [legendCol]="false" for an auto-flowing multi-column layout.

Special Day Properties

  • specialDayNumber: Number 1-20 determining the marker color/style
  • rule: Function that returns true for dates to mark
  • legendText: Text displayed in the legend
  • appointment: (Optional) If true, shows a small circle marker
