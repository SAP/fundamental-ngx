Simple Datetime Picker The simplest form of the datetime picker. It makes use of ngModel to bind to the desired value. The alternative is to use the date input combined with the dateChange output. There is also added [placement] attribute which allows us to decide where popup should be shown The value for the input only changes the model on blur or an enter/return keystroke. Some default options that belong to the time component such as [displaySeconds], [displayMinutes], [displayHours] and [meridian] are based on the DateTimeFormats.display.dateTimeInput options.
For example if Datetime picker display format option contains hours, minutes and day period info then default options will be the next: [displaySeconds]="false", [displayMinutes]="true", [displayHours]="true" and [meridian]="true". The listed above options are applied to the time component only (used internally) and can be defined explicitly if needed. Programmatic Change It is possible to programmatically change the value of the date of the datetime picker and still conserve full functionality. Null Validity Null input values are considered valid by default. Use [allowNull]="false" to make them invalid. Formatting Providing a custom format for the dates is possible through providing DATE_TIME_FORMATS config. Note that when providing a custom format, the [placeholder] input should be set to match the new format as well. The dateTimeFormat pipe can also be utilized to convert the date into a string according to the provided format. Disabled State Date Time Picker in Reactive Form The date-time-picker component may also be used within Angular Reactive Forms. DateTime Picker follows Date Picker with [disableFunction] i18n example.

In order to change the locale of the DatetimePicker, both the calendar will need to set its locale via the DatetimeAdapter and the remaining pieces of the component (such as OK/Cancel buttons, "Hrs/Min/Sec" labels) will need to be translated via the fundamental-ngx/i18n module. This is because the CalendarComponent utilizes different localization strategies than the other components of the library. Note the TypeScript example code below to see how this is done.

Date Time picker that updates it's value on blur. Set [processInputOnBlur]="true" in order to move the processing of input value to whenever component loses focus. You may also hit "enter" to apply the value Mobile mode To use Datetime picker in mobile mode:
  • Set [mobile] property to true
  • Set [mobileConfig] with MobileModeConfig configuration object to customize the look of the control
  • Set [mobileLandscape]="true" to render Calendar in Landscape mode, or [mobilePortrait]="true" for a Portrait mode.
Datetime picker Mobile Mode can be configured globally by providing MOBILE_MODE_CONFIG token. Check Global Configuration page for more information.
