Notification Notifications display information to users about events or actions. The component automatically handles accessibility: aria-describedby is set on action buttons inside fd-notification-body, linking them to the notification header for screen reader context. Template as Content The NotificationService allows passing in a template reference as the content of the notification. Add let-notification to the template element to access the injected NotificationRef.

The example also features a confirmation notification. Subscribe to afterClosed to access the returned value. A dismissal will trigger an error on the observable, similar to rejecting a promise. The example notification is dismissed after 6 seconds.

To allow the notification to stay open on route navigation, set the closeOnNavigation config option to false. Component as Content A component type can be passed to the NotificationService open method as the content of the notification. Inject NotificationRef inside the content component to access methods such as close and dismiss. Notification Group Notification groups organize related notifications under collapsible headers. The group automatically handles accessibility attributes:
  • aria-controls is automatically set on the group header, linking it to the group list for screen readers.
  • The group header supports keyboard navigation with Enter and Space keys to toggle expansion.
Use [expanded]="true" on fd-notification-group to set the initial expanded state.
