FdDate (Built-in) FdDate is the built-in date model. It works out of the box with no extra dependencies — just add provideDateTimeFormats() to your providers.

For advanced needs like locale-aware formatting, UTC handling, or custom parse/display formats, use the Dayjs adapter instead (see below). Dayjs Adapter The Dayjs adapter lets you use Day.js objects directly with date/time components. It supports locale switching, custom formats, UTC mode, and strict parsing.

For application-wide setup, add provideDayjsDatetimeAdapter() to your app or route providers. The example below shows the equivalent manual approach with explicit DatetimeAdapter and DATE_TIME_FORMATS providers — useful when you only need it in a specific component. Switching Locale Call setLocale() on the adapter to change the language. Every picker that shares that adapter instance updates together.

If you need a picker to stay in a different locale, give it a separate adapter by adding DatetimeAdapter to that component's providers. That picker becomes independent.

Try it: Change the locale at the top — both pickers update. Then change the locale at the bottom — only the isolated picker changes.

Remember to import the corresponding dayjs locale (e.g. import 'dayjs/locale/fr') before switching to it. UTC & Strict Mode By default, dates use the local time zone and the parser accepts loosely formatted input. You can change both behaviours with DAYJS_DATE_TIME_ADAPTER_OPTIONS:

UTC mode (useUtc: true) — all dates are stored and displayed in UTC. Useful when every user should see the same date regardless of time zone.

Strict mode (strict: true) — input must match the expected format exactly. "3-1-2026" would be rejected when the format is "MM/DD/YYYY". Custom Formats The DATE_TIME_FORMATS token controls how dates are parsed from user input and displayed back. It has two groups:

parse — how user input is read. For example, dateInput: 'l' accepts the locale's short date format.

display — how dates appear in the input field and to screen readers. For example, dateA11yLabel: 'YYYY MMMM DD' announces "2026 March 12".

The first picker below uses the global format defined in providers. The second picker uses [customDateTimeFormat] to override the format per component — select a format from the dropdown to see it update in real time.
