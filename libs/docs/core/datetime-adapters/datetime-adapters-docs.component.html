FdDate Quick Start The built-in FdDate adapter requires no extra dependencies. Provide it with provideDateTimeFormats() from @fundamental-ngx/core/datetime. Dayjs Quick Start The Dayjs adapter gives you full control over parse and display formats. At the application level, use provideDayjsDatetimeAdapter() from @fundamental-ngx/datetime-adapter. For component-level providers (as shown below), provide DatetimeAdapter and DATE_TIME_FORMATS manually. Locale Switching Switch locales at runtime by calling datetimeAdapter.setLocale(). Remember to import the corresponding dayjs locale (e.g. import 'dayjs/locale/fr'). UTC & Strict Mode By default the DayjsDatetimeAdapter creates dates in your local time zone and accepts loosely formatted input (e.g. "3-1-2026" would be accepted even if the expected format is "03/01/2026").

UTC mode (useUtc: true) — stores and displays dates in Coordinated Universal Time instead of the user's local time zone. Useful when all users should see the same date regardless of where they are.

Strict mode (strict: true) — rejects input that does not exactly match the expected format. For example, "3-1-2026" would be invalid when the format requires "MM/DD/YYYY". This helps catch typos and ambiguous dates early.

Provide DAYJS_DATE_TIME_ADAPTER_OPTIONS to enable either or both options. Custom Formats The DATE_TIME_FORMATS token tells date/time components how to read user input and how to show dates back to the user. For example, you can decide whether dates appear as 3/12/2026 (US short format) or 2026-03-12 (ISO format).

The token has two groups of format strings:

parse — how the component reads what the user types. For instance, dateInput: 'l' means "accept the locale's short date format" (e.g. 3/12/2026 in English).

display — how the component shows dates in the input field and to screen readers. For instance, dateA11yLabel: 'YYYY MMMM DD' makes the screen reader announce "2026 March 12".

Moment Adapter (Deprecated) If your project still uses Moment.js, you can use the MomentDatetimeAdapter from @fundamental-ngx/moment-adapter. It works the same way as the Dayjs adapter — provide DatetimeAdapter and DATE_TIME_FORMATS at the component or application level.

Note: This adapter is deprecated. Moment.js is now considered legacy by its maintainers. We recommend migrating to the Dayjs adapter (see the next section). 1. Install the Dayjs adapter: npm i @fundamental-ngx/datetime-adapter dayjs
