Basic This example shows the minimum shellbar for a single application product with only user settings. If no user thumbnail is available then display initials. Links with collapsible menu and product switcher. The shellbar can have 4 different sizes 's' | 'm' | 'l' | 'xl', where "m" is default one.
When a product has multiple links, the product links should collapse into an overflow menu on mobile screens (s size). All actions, except for the user menu, should be collapsed.
ProductSwitchComponent and ProductMenuComponent have properties straight from popover component, so some customisation can be added, by passing inputs ex. [position]="'bottom-end'". Shellbar with Select The select component can also be used with shellbar. Responsive Shellbar Responsive Shellbar Shellbar with SideNavigation When Shellbar is used together with Vertical Navigation (aka Side Navigation) set the sideNav property to true. This will remove the responsive paddings applied on different screen sizes and will align vertically the control button with the items in the Side Navigation. The control button, situated on the far left hand-side of the Shellbar, is responsible for switching the modes of the Side Navigation (expanded, condensed and off-screen) and requires the fd-shellbar-side-nav directive. Shellbar with Responsive SideNavigation The Side Navigation can be set to collapse automatically at a width breakpoint specified by the developer via the [collapseWidth] input. In this example the breakpoint has been set at 768 pixels. How it works
  • The shellbar should be placed inside the shell layout container.
  • The shellbar is fluid and responsive by default with proper margins and padding built in.
  • The shellbar actions are duplicated into the overflow menu on mobile screens.
  • Product menu data gets passed as inputs to the <fd-product-menu> component. The user, user menu and product switcher are passed as inputs to the <fd-shellbar-actions> component. Data for each action gets passed as inputs to the <fd-shellbar-action> component. The response to clicks on any of these items is handled via callback functions for each object. See the 'collapsible' example to gain an understanding of how this works.
