Default tree

Tree Component allows the user to display and work with a hierarchical set of items.

Tree Component supports two approaches of rendering the nodes:

  • Declarative: works with content projected fd-tree-item components. Child tree items should be inside parent tree item;
  • Imperative: works with data source and provided fd-tree-item definition with the help of *fdTreeItemDef directive

Tree item text is rendered with the help of fdTreeItemText, which allows developers to use any type of content as a text (for example, TemplateRef).

Tree item icon can be added with the help of fdTreeItemIcon directive.

Keyboard navigation: Tree component supports keyboard navigation with the help of arrow keys: up and down will navigate through the items on the same level, left and right will navigate to child or parent level depending on content direction.

Navigatable tree items

Tree items can be marked as navigatable. By passing [navigatable]="true" input property, tree item will become interactable.

In order to render nafvigation indicator, developers should pass input property [navigationIndicator]="true" to the Tree component.

Highlight indicators

Tree item component supports applying highlight indicators.

In order to set the indicator, developers need to pass [state] input property with appropriate value.

Tree item action buttons

Tree item component supports projecting action buttons.

In order to add such buttons, developers should pass element with fdTreeItemButton directive to the tree item content.

Selectable tree items

Tree supports selectable state for the tree items.

To enable selection, add [selection]="'single' | 'multiple'" input property to the fd-tree.

Additionally, developers can specify the location of selection indicator with the help of [selectionPlacement] input property.

Selectable tree can also work with Angular Forms. Both reactive and template driven.

 Lazy loading tree items

Tree component can work with Arrays, Observables and FdTreeDataSource.

Even though the component does not directly consume child items until the parent item being expanded, passing a large array directly may cause performance issues.

To avoid that, we recommend passing child items as a separate instance of the Data Source, which will load the data only when it's necessary.
