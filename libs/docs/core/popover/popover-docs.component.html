Basic Popovers Making use of simple popovers is very easy. Open an fd-popover component, place a control and body inside. You are not required to keep track of the popover state. Simple Popover There is different way to build popover, it can be done, by using simplified markup, with trigger element connected to popover. Popover with Header and Footer Popover body can be created with header and footer components. To add header, there can be added fd-popover-body-header component, for footer there is dedicated fd-popover-body-footer component. Popover Placement Popover offers a variety of placements for users to choose from. These can be any of 'top', 'bottom', 'right', 'left', and then can be suffixed with '-start' or '-end'. Programmatic Control Controlling the popover through programmatic means is easy. Simply two-way bind the isOpen property and change it. There are also some methods such as open() and close() which can be called through reference variable. Popovers in Dialogs Popovers are functional inside dialogs. Fill Control Width Use fillControlMode input property to control when popover should fill control width. Different Popover Container Different element to which the popover body is attached can be added, by passing [appendTo]="elementRef". Otherwise the popover body is rendered out of markup, in < body >. Popover Focus Trap To apply focus trap, you need to add [focusTrapped]="true". To move focus automatically into popover, when it's opened, you need to apply [focusAutoCapture]="true". After closing popover, focus will go to latest active element. Popover CDK Placement There is different way of putting placement, which is accepted by CdkOverlay. [cdkPositions] is the property that accepts array of objects with positions related to overlay(popover-body) and origin(popover-control).
  • For horizontal (overlayX, originX), those values are 'start' | 'center' | 'end'.
  • For vertical (overlayY, originY), those values are 'bottom' | 'center' | 'top'.


This property accepts array, just to deal with situation, when overlay body goes beyond the screen. Popover Scroll Example There is also way to change scroll behaviour, CDK Overlay offers 4 different options
  • CloseScrollStrategy - popover is closed, upon scrolling.
  • NoopScrollStrategy and BlockScrollStrategy - popover stays in same place.
  • RepositionScrollStrategy(default) - popover follows the element, to which is appended to. There is way to apply another argument, to close popover, when it goes beyond the screen and add throttle time for performance purposes.
Dynamic Body Height In case of height change inside popover body, the position of popover is not automatically changed, mostly cause of optimization reasons. To force change of the position, it is needed to call updatePopover() method inside popover, as it is on example. There is used SideNavigation component. Dropdown Popover Dropdown is a composition of basic popover and button with [fdMenu]="true". Responsive Popover in mobile mode To use Popover in mobile mode:
  • Set [mobile] property to true
  • Add <ng-template #popoverBodyContent>...<ng-template> inside fd-popover to pass popover body content from outside
  • Add <ng-template #popoverFooterContent>...<ng-template> inside fd-popover to pass popover footer content from outside
Popover Mobile Mode can be configured globally by providing MOBILE_MODE_CONFIG token. Check Global Configuration Page for mobile information. Handling dynamic control offset

There might be cases when offset of the popover control is changed dynamically without user interaction.

For this case, you can add fdPopoverContainer directive to the parent element.

When the size of the element with fdPopoverContainer changes, it will automatically trigger position refresh of all opened popovers in it.
