Skeleton
Skeleton is the replacement for the old spinner loading component. Using skeleton you can show a placeholders of a certain shape while data is being loaded.

The idea behind skeleton consists of two parts:

Providing state

  • Globally. Import skeleton module in your root module using SkeletonModule.forRoot() with optionally passing default skeleton state. Then inject SkeletonGlobalService and use setSkeletonState() to change the global skeleton state.
  • Locally. Use fdSkeletonState template directive and pass either boolean or 'global' value to it to set skeleton state for the elements subtree. The 'global' value means that subtree reacts only to the global skeleton state changes.

Consuming State, Showing Placeholders

  • Single placeholder. Use fdSkeletonConsumer directive on the element level or add a providers for the directive/component using skeletonConsumerProviders() factory function.

    You can pass additional configuration using directive's input properties or by passing an object to the factory function.

  • Multiple placeholders. Use fdSkeletonTemplate template directive to define template that should be rendered instead of a real content.

Additionally, to create skeleton templates you can use SkeletonComponent component (included in SkeletonModule).
