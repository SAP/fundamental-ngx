ContentDensityStorage

ContentDensityModule has support for configurable storage of application content density preference. Out of the box, 3 storage options are available:

  • LocalContentDensityStorage - mode gets saved and consumed in local storage
  • UrlContentDensityStorage - mode gets saved and consumed from URL
  • MemoryContentDensityStorage - mode is saved in memory

The documentation page uses LocalContentDensityStorage, so if you change content density preference from either toolbar or from GlobalContentDensityService example, changes will be saved in local storage

Also, as an additional comfort, you can modify under which key does storage get saved in local storage or in url query params

GlobalContentDensityService Content Density can be maintained app-wide using the GlobalContentDensityService, and components can subscribe to a Observable managed by the service and use that to handle changes to the content density. Before using it, you should add ContentDensityModule to imports array of AppModule, or other root module in your application.
Note that many components only come in one size, and others (such as the button) only have compact and cozy sizes, so setting the size to condensed will have same effect as compact.
All components that have content density options will first check to see if the developer has provided a fdContentDensity directive, and if not, it will then refer to the value set in the GlobalContentDensityService. Directives

You can use directives to make components explicitly use desired content density configuration. We have created one directive called [fdContentDensity], which also has 3 other names [fdCompact], [fdCozy] and [fdCondensed]. Although these three names are just for ease of use and writing less code.
