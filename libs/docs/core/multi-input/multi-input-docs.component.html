Simple Multi Input The standard version of the component that most people will use regularly. Width of the input can be controlled simply through the top-level component element sizing. Overflowing tokens can be reviewed by swiping the tokens left and right. Compact Multi Input The compact Multi Input is for large screens. The user can review tokens by pressing the right or left arrows on the keyboard. Mobile Mode Multi Input To use Multi Input in mobile mode:
  • Set [mobile] property to true
  • Set [mobileConfig] with MobileModeConfig configuration object to customize the look of the control
Multi Input Mobile Mode can be configured globally by providing MOBILE_MODE_CONFIG token. Check Global Configuration page for more information. Display Object Property The input supports objects through the displayFn input. The way the object is displayed to the user is fully customizable through providing a function that accepts the object and returns the string to be displayed. The example below shows displaying an object field in full uppercase. Return results including search term By default, the multi-input will only return results where the result starts with the search term. By setting [includes]="true", the multi-input will return results where the result includes the search term at any place in the string. Custom Filter The input supports custom filters through the filterFn input function. Simply pass a function which accepts an array and a search term, and then returns the desired filtered array. In this example, we are requiring that the search term is case-sensitive. Observable Async Example The dropdownValues input can accept an Observable with the async pipe, which could be useful for autocomplete. Multi Input in Reactive Form The multi-input component may also be used within an angular reactive form. Adding New Tokens Set [allowNewTokens] to "true" to allow the creation of new tokens in the multi-input. Note that if you are populating the multi-input with objects and using a custom displayFn, you will also need to implement your own newTokenParseFn so your multi-input knows how to format a new object. This function is not required if your collection is simply an array of strings. To customise validation function, you can pass [newTokenValidateFn], which by default doesn't allow empty strings. Custom Item Template The way the items are rendered in the popover list can be customized by providing an itemTemplate to the component. The itemTemplate should align with the contents of an fd-list-item directive. Predefined dropdown width

By default, Multi Input component doesn't limit the width of the dropdown.

This means that the dropdown will grow depending on the inner items width.

Developers can change this behaviour with two options: by providing width of the dropdown in pixels, or by setting the max width of the dropdown equal to the Multi Input component itself.

Addon Button Clicked Event

The addOnButtonClicked event is emitted when the user clicks the addon button. This event can be captured to further configure custom behavior.
