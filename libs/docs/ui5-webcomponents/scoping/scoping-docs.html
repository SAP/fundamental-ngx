Overview

UI5 Web Components are registered as custom elements in the browser. The custom element registry is global and allows only one definition per tag name. This becomes a problem when multiple micro-frontends or applications on the same page ship different versions of UI5 Web Components — they would try to register <ui5-button> twice.

Scoping solves this by appending a suffix to every tag name. For example, with the suffix myapp, the tag <ui5-button> becomes <ui5-button-myapp>. Each application gets its own isolated set of custom element definitions.

When do you need scoping?

You need scoping when different versions of UI5 Web Components run on the same page. This typically happens in:

  • Micro-frontend architectures (Module Federation, single-spa, iframe-less shell apps)
  • Multiple Angular apps composed into a single page

If all apps on the page use the same version of UI5 Web Components, they share the same custom element registration and scoping is not needed.

Setup

Add this call at the very top of your main.ts, before any UI5 component import. The suffix must contain only alphanumeric characters, dashes, and underscores (/^[a-zA-Z0-9_-]+$/). 

import { setCustomElementsScopingSuffix } from '@ui5/webcomponents-base/dist/CustomElementsScope.js';

setCustomElementsScopingSuffix('myapp');

Order matters. setCustomElementsScopingSuffix must be called before any UI5 Web Components are imported or registered. If called too late, a console warning will appear and some components may not be scoped.

That's it. No other configuration is needed — UI5 handles the rest internally.

 Using scoped components in Angular templates

Once scoping is active, all UI5 custom element tags get the suffix appended (e.g. <ui5-button> becomes <ui5-button-myapp>). The component behavior, properties, events, and slots remain identical — only the HTML tag name changes.

Use the scoped tag name as the HTML element and add the original name as an attribute to activate the Angular wrapper: 

<!-- With suffix "myapp" -->
<ui5-button-myapp ui5-button [design]="'Emphasized'">Submit</ui5-button-myapp>
<ui5-input-myapp ui5-input [placeholder]="'Search...'"></ui5-input-myapp>
<ui5-dialog-myapp ui5-dialog [headerText]="'Confirm'">...</ui5-dialog-myapp>

Both parts are required: the scoped tag (ui5-button-myapp) registers the custom element, and the attribute (ui5-button) activates the Angular wrapper.

Handling Angular's unknown element warnings

When you use scoped tag names like <ui5-button-myapp>, Angular does not recognize them as known elements by default. To suppress the NG8001 warnings, add CUSTOM_ELEMENTS_SCHEMA to your component: 

import { CUSTOM_ELEMENTS_SCHEMA, Component } from '@angular/core';

@Component({
    selector: 'app-my-feature',
    template: `<ui5-button-myapp ui5-button [design]="'Emphasized'">Submit</ui5-button-myapp>`,
    schemas: [CUSTOM_ELEMENTS_SCHEMA],
    imports: [Button]
})
export class MyFeatureComponent { }

This tells Angular to allow any unknown HTML element without throwing a compile error.

 Advanced: Scoping rules

By default, all tags starting with ui5- are scoped. You can customize which tags get scoped using setCustomElementsScopingRules: 

import {
    setCustomElementsScopingSuffix,
    setCustomElementsScopingRules
} from '@ui5/webcomponents-base/dist/CustomElementsScope.js';

// Set the suffix
setCustomElementsScopingSuffix('myapp');

// Only scope tags starting with "ui5-" but exclude "ui5-icon"
setCustomElementsScopingRules({
    include: [/^ui5-/],
    exclude: [/^ui5-icon$/]
});

include — array of regular expressions. A tag must match at least one to be scoped.
exclude — array of regular expressions. A tag matching any exclude rule is not scoped, even if it matches an include rule.

CSS considerations

If you have global CSS rules targeting UI5 tag names, they need to be updated to match the scoped names:

/* Before scoping */
ui5-button {
    margin-right: 8px;
}

/* After scoping (suffix: "myapp") */
ui5-button-myapp {
    margin-right: 8px;
}

/* Or use attribute selectors to work with any suffix */
[ui5-button] {
    margin-right: 8px;
}

Using the attribute selector [ui5-button] in CSS is a good strategy because it works regardless of whether scoping is enabled and what suffix is used.

Complete example

Putting it all together for a micro-frontend called orders:

main.ts

// main.ts — scoping FIRST, before anything else
import { setCustomElementsScopingSuffix } from '@ui5/webcomponents-base/dist/CustomElementsScope.js';
setCustomElementsScopingSuffix('orders');

(async () => {
    const { bootstrapApplication } = await import('@angular/platform-browser');
    const { appConfig } = await import('./app/app.config');
    const { App } = await import('./app/app');

    await bootstrapApplication(App, appConfig);
})().catch((err) => console.error(err));

order-list.ts

import { CUSTOM_ELEMENTS_SCHEMA, Component } from '@angular/core';
import { Button } from '@fundamental-ngx/ui5-webcomponents/button';
import { Input } from '@fundamental-ngx/ui5-webcomponents/input';
import { Table } from '@fundamental-ngx/ui5-webcomponents/table';

@Component({
    selector: 'app-order-list',
    templateUrl: './order-list.html',
    schemas: [CUSTOM_ELEMENTS_SCHEMA],
    imports: [Button, Input, Table]
})
export class OrderList {
    onSearch(event: Event): void {
        // handle search
    }
}

order-list.html

<!-- All tags use the scoped name + attribute selector -->
<ui5-input-orders ui5-input
    [placeholder]="'Search orders...'"
    (ui5Input)="onSearch($event)">
</ui5-input-orders>

<ui5-table-orders ui5-table>
    <!-- table content -->
</ui5-table-orders>

<ui5-button-orders ui5-button [design]="'Emphasized'">
    New Order
</ui5-button-orders>
 Troubleshooting

Console warning: "Setting the scoping suffix must be done before importing any components"

setCustomElementsScopingSuffix was called after a UI5 component was already imported. Move the call to the very first lines of main.ts, before any other imports that might trigger component registration.

Component does not render (empty element)

Make sure the HTML tag uses the scoped name (e.g. <ui5-button-myapp>), not the original name (<ui5-button>). With scoping active, the original tag name is not registered in the custom element registry.

Angular input/output bindings not working

Verify that the ui5-button (or equivalent) attribute is present on the element. The Angular wrapper needs the attribute to activate. Also check that the wrapper component is listed in the imports array of your component.

NG8001: Unknown element warning

Add CUSTOM_ELEMENTS_SCHEMA to the schemas array of your component decorator. See the Handling Angular's unknown element warnings section above.
