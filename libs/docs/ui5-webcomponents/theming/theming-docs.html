Overview

Call provideUi5ThemingBridge() once and fundamental-ngx theme changes automatically propagate to UI5 Web Components — both light/dark switches and high-contrast modes.

Setup

Two steps are required:

Step 1: Register UI5 theming assets in main.ts (before Angular bootstrap):

// main.ts
import '@ui5/webcomponents-theming/dist/Assets.js';

This registers the CSS variable sets for all SAP themes. Without it, setTheme() calls inside UI5 won't find the theme data. The assets are loaded lazily on first use — no upfront bundle size impact.

Step 2: Add the theming bridge to your app providers:

// app.config.ts
import { provideTheming, themingInitializer } from '@fundamental-ngx/core/theming';
import { provideUi5ThemingBridge } from '@fundamental-ngx/ui5-webcomponents-base/theming';

export const appConfig: ApplicationConfig = {
  providers: [
    provideTheming({ defaultTheme: 'sap_horizon' }),
    themingInitializer(),
    provideUi5ThemingBridge(),
  ]
};

Now switching themes via ThemingService.setTheme(), the toolbar selector, or the ?theme= URL parameter updates both fundamental-ngx and UI5 components together.

How it works

The data flow:

ThemingService.currentTheme   (Observable, set by provideTheming + init)
        |
        |  Ui5ThemingBridgeService subscribes
        v
Ui5ThemingService.setTheme()  (calls UI5's setTheme — shadow DOM re-renders)

Ui5ThemingBridgeService is an Angular service activated by provideUi5ThemingBridge(). It subscribes to the core ThemingService.currentTheme Observable and calls Ui5ThemingService.setTheme() whenever the theme changes. This is explicit opt-in — apps not using UI5 components don't pay for this bridge.

The bridge handles both initial theme (from URL ?theme= parameter or config default) and runtime theme switches. It uses takeUntilDestroyed() for automatic cleanup.
