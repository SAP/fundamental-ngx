Basic Sample The Dialog component is a modal popup that displays content in a layer above the app. Dialogs are used to prompt users for input, confirmation, or to display important information that requires user attention. The dialog blocks interaction with the page content until it is dismissed. Header and Footer Dialogs support custom header and footer content using slots. Use the header slot for custom header content (like titles with icons), and the footer slot for action buttons. You can also use the simple headerText property for basic text headers. Draggable and Resizable Dialogs can be made draggable by setting the draggable property, allowing users to move them by the header. They can also be resizable, which adds a resize handle in the bottom corner. These features are only available in desktop mode. Value States Dialogs support different states to communicate the severity or type of information: None (default), Positive, Negative, and Critical. When Negative or Critical is set, the accessibility role changes to "alertdialog" for screen readers. Stretch to Full Screen Use the stretch property to make the dialog fill the entire screen on mobile devices, or approximately 90% of the viewport on desktop. This is particularly useful for forms or content that requires more space on smaller screens. Events Dialogs emit lifecycle events: ui5BeforeOpen and ui5BeforeClose (both cancellable), and ui5Open and ui5Close (after the action completes). Use these events to perform actions when the dialog state changes or to prevent opening/closing based on validation.
