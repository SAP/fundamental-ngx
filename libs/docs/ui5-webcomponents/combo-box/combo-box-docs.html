Basic ComboBox The ui5-combobox component allows users to select a value from a predefined list or enter a custom value. It combines the functionality of a dropdown and an input field. Use ui5-cb-item components as children to define the available options. The ui5Change event fires when the value is changed. Value States The valueState property provides visual feedback about the input validation. Available states include None (default), Positive, Critical, Negative, and Information. Each state displays a different color and icon to indicate the validation status. Filter Types The filter property controls how items are filtered when the user types. Available options include StartsWithPerTerm (default - matches items where the search value appears at the start of the text or at the start of any word within the text, eg. check the string "Britain"), StartsWith (matches items starting with typed text), Contains (matches items containing typed text), and None (no filtering). Component States The ComboBox supports various states: disabled (completely non-interactive), readonly (displays selected value but cannot be changed), required (indicates required field), and loading (shows loading indicator in the picker). Additional Text Use the additionalText property on ui5-cb-item to display supplementary information next to the main text. This is useful for showing codes, descriptions, or other contextual information alongside the primary item text. Clear Icon The showClearIcon property enables a clear icon in the input field, allowing users to easily clear the selected value. Items Grouping The ui5-cb-item-group component allows you to group items within the ComboBox, making it easier for users to find related options.
