Basic Multi-Input The multi-input component allows users to enter multiple values as tokens. Users can type to search and select from suggestions, and each selected value appears as a token that can be individually removed. Use the ui5-token component in the tokens slot to display selected values, and ui5-suggestion-item components to provide autocomplete suggestions. Value States Multi-input supports different value states to provide visual feedback about validation or status:
  • None (default) - Standard appearance
  • Positive - Indicates success or valid input
  • Critical - Indicates a warning condition
  • Negative - Indicates an error or invalid input
  • Information - Provides informational context
Use value states to communicate validation results or provide contextual feedback to users. Disabled and Read-only States Multi-input can be disabled or readonly:
  • Disabled: Completely non-interactive, cannot receive focus.
  • Read-only: Not editable and new tokens cannot be added, but the component can receive focus.
Event Handling Multi-input emits various events for user interactions:
  • ui5Input - Fired when the value changes at each keystroke
  • ui5Change - Fired when the input operation has finished (Enter or focus out)
  • ui5TokenDelete - Fired when tokens are being deleted
  • ui5SelectionChange - Fired when navigating through suggestions
  • ui5Open - Fired when the suggestions picker opens
  • ui5Close - Fired when the suggestions picker closes
Use these events to handle user input and implement custom logic.
