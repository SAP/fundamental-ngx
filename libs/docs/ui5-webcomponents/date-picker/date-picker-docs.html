Basic Sample The Date Picker component allows users to select a date from a calendar popup. It provides keyboard navigation, accessibility support, and internationalization capabilities. The selected date is displayed in the input field according to the specified format pattern. Date Format Patterns Date Picker supports various format patterns for displaying dates. Use formatPattern to control how dates appear in the input field. Common patterns include yyyy-MM-dd, dd/MM/yyyy, MM/dd/yyyy, and localized patterns. You can also use displayFormat and valueFormat to control the display and value formats separately. Min and Max Dates

Use minDate and maxDate properties to restrict the date selection range. Dates outside this range will be disabled in the calendar popup.

Note: When using minDate and maxDate, ensure the date format matches your formatPattern or use ISO format (yyyy-MM-dd).

Value States Date Picker supports different value states to indicate validation status: None, Positive, Critical, Negative, and Information. Each state displays a different color and icon to communicate the field's status to users. Component States Date Picker can be disabled, readonly, or required. Use these properties to control user interaction and indicate mandatory fields. Calendar Types Date Picker supports different calendar types including Gregorian, Islamic, Japanese, Buddhist, and Persian. Use primaryCalendarType and secondaryCalendarType to display one or two calendar types simultaneously.
