Overview

Set FD_LANGUAGE_SIGNAL once and the bridge keeps fundamental-ngx translations, locale formatting, and UI5 component language in sync automatically.

Setup

Two steps are required:

Step 1: Register UI5 locale assets in main.ts (before Angular bootstrap):

// main.ts
import '@ui5/webcomponents-localization/dist/Assets.js'; // CLDR data (date/number formatting)
import '@ui5/webcomponents/dist/Assets.js';              // Component i18n (tooltips, labels, aria-text)
// If using Fiori components:
// import '@ui5/webcomponents-fiori/dist/Assets.js';

Without these imports, translated strings (tooltips, calendar labels) stay in English after a language switch. The actual locale data is loaded lazily on first use — no upfront bundle size impact.

Step 2: Add the language bridge to your app providers:

// app.config.ts
import {{ '{' }} provideUi5LanguageBridge {{ '}' }} from '@fundamental-ngx/ui5-webcomponents-base/i18n';

export const appConfig: ApplicationConfig = {{ '{' }}
  providers: [
    provideUi5LanguageBridge(),
  ]
{{ '}' }};

Now langSignal.set(FD_LANGUAGE_GERMAN) updates all fundamental-ngx and UI5 components together.

 How it works

The signal flow:

FD_LANGUAGE_SIGNAL  (set by your app)
        |
        |  linkedSignal derives locale
        v
FD_LOCALE_SIGNAL    ('de', 'fr', 'ja', ...)
        |
        |  Ui5LanguageService effect()
        v
UI5 setLanguage()   (UI5 components re-render)

Ui5LanguageService is an Angular service activated by provideUi5LanguageBridge(). It watches FD_LOCALE_SIGNAL via an effect() and calls UI5's setLanguage() whenever the locale changes. This is explicit opt-in — apps not using UI5 components don't pay for this bridge.
