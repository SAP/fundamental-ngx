Basic Usage

@fundamental-ngx/i18n exports pre-translated messages for {{ supportedLanguages }}. To use them, import FD_LANGUAGE_{{ '{' }}LANGUAGE_NAME{{ '}' }} (e.g., FD_LANGUAGE_UKRAINIAN) and provide it using the signal-based token FD_LANGUAGE_SIGNAL.

Provide translations at the application level in app.config.ts to make them available throughout your app, or at the component level for component-specific translations (as shown in the example below).

Note: The old FD_LANGUAGE (Observable) token is deprecated and will be removed in a future version. Use FD_LANGUAGE_SIGNAL instead.

Loading from JSON files

You can load translations dynamically from JSON files at runtime. This is useful for:

  • Loading custom translations without bundling them in your application code
  • Supporting additional languages beyond the built-in ones
  • Allowing non-developers to manage translation files
  • Enabling dynamic language switching based on user preferences

Use the loadJson helper function to transform a JSON object into the FdLanguage interface. The JSON keys should use dot notation (e.g., "coreDatePicker.dateInputLabel").

Loading from .properties files

You can load translations from .properties files, which are commonly used in Java-based internationalization systems and SAP applications.

Use the loadProperties helper function to parse the properties file format into the FdLanguage interface. Each line follows the format: key = value.

Benefits of .properties files: Simple text format, easy to edit, widely supported by translation tools, and commonly used in enterprise applications.
