Patching translations (partial overrides)

Sometimes you need to override just a few translation keys for a specific component or section of your app, rather than replacing the entire language. This is called patching or partial override.

Common use cases for patching:

  • Customizing specific messages for a particular component or page
  • Providing domain-specific terminology (e.g., "Customer" instead of "User")
  • A/B testing different wording in specific areas
  • Overriding a few translations without maintaining a full language file

Two approaches for patching translations:

  1. Using fdPatchLanguage directive: Apply partial overrides directly to a specific component in the template. Best for one-off customizations.
  2. Using patchLanguage() provider function: Provide partial overrides in a component's providers array. Best for wrapping multiple child components or creating reusable translation scopes.
