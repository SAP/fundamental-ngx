Writing translations

Translations in fundamental-ngx consist of translation keys (identifiers) and translation values (the actual text displayed to users). Understanding how to write effective translations is essential for internationalizing your application.

Translation values can be:

  • Simple strings - Plain text without any dynamic content
  • Strings with placeholders - Text with dynamic values using {{ '{' }}paramName{{ '}' }}
  • ICU Message Syntax - Advanced formatting for pluralization, gender, date/time using ICU expressions
  • Internal references - Reusing other translations with {{ '{' }}@@keyName{{ '}' }}
  • Functions - JavaScript functions for complex logic (when loading from JS objects)

Internal references (reusing translations)

Internal references allow you to reuse existing translations within other translations. This helps maintain consistency and reduces duplication. Use the syntax {{ '{' }}@@translationKey{{ '}' }} to reference another translation.

How it works:

  1. The translation resolver detects the @@ prefix
  2. It looks up the referenced translation key
  3. If found, it replaces the reference with the resolved translation
  4. The complete string is then processed through ICU Message Syntax interpolation

Important: If a referenced translation requires parameters (context), the referencing translation must also provide those same parameters. For example, if platformTable.headerMenuFreezePlural needs count, then platformTable.messageNoDataWithFilter must also receive count in its context.

Warning: Internal references create tight coupling and maintenance risks. Avoid using this feature when possible. Never reference translations from other components — if the referenced component is deprecated or removed, your translations will break silently at runtime. If you must use internal references, keep them within the same component and only for simple, reusable units like pluralization rules. Escaping curly braces

Since ICU Message Syntax uses curly braces {{ '{' }} and {{ '}' }} for parameters and expressions, you need to escape them if you want to display them as literal characters.

Escaping rules:

  • Wrap curly braces in single quotes {{ "'" }}{{ '{' }}{{ "'" }} or {{ "'" }}{{ '}' }}{{ "'" }}
  • Or use double quotes {{ '"' }}{{ '{' }}{{ '"' }} or {{ '"' }}{{ '}' }}{{ '"' }}
