Basic Notification List The notification list component displays a list of notifications to users. Each notification is represented by a ui5-li-notification component that contains a title, description, and optional avatar. Users can click on notifications to view details or close them using the close button. Use the ui5ItemClick event to handle notification clicks and ui5ItemClose to handle notification dismissals. States and Importance Levels Notification list items support different states and importance levels to provide visual feedback:
  • States: None, Positive, Critical, Negative, and Information - Use states to indicate the semantic meaning or urgency of the notification
  • Importance: Standard and Important - Use importance levels to prioritize notifications and help users identify which ones require immediate attention
Read and Unread States Notifications can be marked as read or unread using the read property. Unread notifications are displayed with bold title text to draw attention. This is useful for helping users track which notifications they've already seen. Click on a notification to mark it as read, or implement a "Mark all as read" feature for bulk operations. Notification Actions Notifications can include action menus using the menu slot. Add a ui5-menu component with ui5-menu-item children to provide contextual actions for each notification. This is useful for workflows like approving requests, viewing details, or taking other notification-specific actions. Group Items The notification list supports grouping of related notifications to provide a more organized view:
  • Group by: Notifications can be grouped by various criteria such as type, sender, or project.
  • Expand/Collapse: Users can expand or collapse groups to manage their view and focus on specific notifications.
Use grouping to enhance the user experience and make it easier to process notifications.
