Basic Side Navigation The Side Navigation component provides a vertical navigation menu with hierarchical structure. Items can have icons and text, and users can select items to navigate. The ui5SelectionChange event is fired when a user selects an item. Use ui5-side-navigation-item components to create navigation items, and set the selected property to indicate the currently active item. Collapsed / Expanded Mode The collapsed property controls whether the side navigation is displayed in collapsed mode (icons only) or expanded mode (icons with text). In collapsed mode, only icons are visible, saving horizontal space. When users hover over items in collapsed mode, tooltips show the item text. Toggle the property to switch between modes, typically using a toggle button in your application header.

Note: The collapsed mode is not supported on phones. The component is intended to be used within a ui5-navigation-layout component to ensure proper responsive behavior. Nested Items with Sub-items

Side navigation items can have nested sub-items using ui5-side-navigation-sub-item components inside ui5-side-navigation-itemcomponents. Parent items can be expanded or collapsed using the expanded property.

When a parent item is marked as [unselectable]="true", clicking on it does not trigger ui5SelectionChange event. Instead, when clicked, it toggles the expansion state (check "Suppliers").

When a parent item is marked as [unselectable]="false" or not marked at all (the default behavior), upon clicking on the text, the ui5SelectionChange event is triggered. In order to toggle the expansion state, the chevron icon needs to be clicked (check "Products").

Sub-items can also be selected independently.

 Fixed Items The fixedItems slot allows you to place navigation items at the bottom of the side navigation. These items remain fixed and are typically used for settings, help, or other utility functions that should always be accessible. Fixed items should be kept "flat" (without sub-items) for the best user experience. Use the same ui5-side-navigation-item component with the slot="fixedItems" attribute. Overlay Mode SideNavigation is enabled to be used inside a responsive popover. This example shows how to achieve overlay mode with correct design and interaction. Custom Width The width of the side navigation panel can be customized using CSS. The panel must be at least 16 rem wide to maintain readability and layout stability. You can set a larger width as needed, but it cannot be reduced below 16 rem. If the panel does not have sufficient space, long texts will wrap automatically.
