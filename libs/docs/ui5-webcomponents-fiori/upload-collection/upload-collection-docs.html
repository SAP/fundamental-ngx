Basic Upload Collection

The Upload Collection component displays a list of uploaded files and allows users to manage them. Each file is represented by a ui5-upload-collection-item that shows the file name, progress, and actions.

Use the ui5ItemDelete event to handle file deletions.

Mark the item as type="Detail" and use the ui5ItemRename event to handle file renaming.

Selection Modes The Upload Collection supports different selection modes to enable file selection and batch operations: None, Single, SingleStart, SingleEnd, SingleAuto, Multiple. Use the selectionMode property to configure the desired selection behavior. The ui5SelectionChange event fires when the selection changes, allowing you to track selected files. Upload States and Progress Upload Collection items support different upload states to provide visual feedback during the upload process:
  • Ready: File is ready to be uploaded, shows progress indicator
  • Uploading: File is currently being uploaded, shows progress indicator and terminate button
  • Error: Upload failed, shows progress indicator and retry button
  • Complete: Upload finished successfully, hides progress indicator
Use the uploadState and progress properties to control the visual state and progress of each file. Handle ui5Retry and ui5Terminate events to implement retry and cancel functionality. Drag and Drop The Upload Collection provides a drag and drop overlay for easy file uploads. When files are dragged over the component, an overlay appears to indicate that files can be dropped. By default, the drag and drop overlay is enabled. You can disable it by setting the hideDragOverlay property to true.

Note: The component only displays the overlay. It's the application developer's responsibility to handle the drop event and implement the actual file upload logic. The drop event is fired only when elements are dropped within the drag and drop overlay area. Custom Header

The Upload Collection supports custom headers through the header slot. You can add a custom header with additional controls like upload buttons, filters, or search functionality. When using a custom header, it's your responsibility to provide proper labeling for accessibility by using the accessibleName property.
