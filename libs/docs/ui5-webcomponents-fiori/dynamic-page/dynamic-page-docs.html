Dynamic Page Dynamic Page component examples with configurable header, content area, and footer functionality Accessibility Attributes

The accessibilityAttributes property allows you to customize ARIA landmark roles and accessible names for each section (root, header, content, footer). This provides fine-grained control over how screen readers interpret the page structure.

Additionally, the ui5-dynamic-page-header now supports the accessibleName property to set its accessible name directly.

Available roles per section:

  • root: none, main, region
  • header: none, banner, region
  • content: none, main, region, form
  • footer: none, contentinfo, region
