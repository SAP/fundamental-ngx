Search with Suggestions The Search component can display suggestions as users type using ui5-search-item components. The noTypeahead property controls whether the value should be autocompleted to match a suggestion item. Suggestions help users find what they're looking for faster by providing relevant options based on their input. Search with Scopes The Search component supports scopes using ui5-search-scope components in the scopes slot. Scopes allow users to filter their search by different categories or areas. For example, users can choose to search in "All", "Products", "Customers", or "Orders". The ui5ScopeChange event is fired when a user selects a different scope, allowing you to adjust the search results accordingly. Loading State The loading property displays a loading indicator in the search popup while fetching results. This provides visual feedback to users that their search is being processed. Use this feature when performing asynchronous searches or API calls. You can also use the messageArea slot to display custom messages or illustrated content when no results are found or during loading. Grouped Items with Show More The Search component can organize suggestions into groups using ui5-search-item-group components. When a group contains many items, you can use the ui5-search-item-show-more component to initially show only a subset of items and allow users to expand the list. The itemsToShowCount property displays the number of hidden items. This example also demonstrates deletable items - users can remove items from the list using the delete button.
